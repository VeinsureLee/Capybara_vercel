# 生命周期

> 用户、卡皮巴拉、旅行、对话会话的完整生命周期设计。

---

## 4.1 用户生命周期

```
┌─────────┐    注册     ┌──────────┐   命名卡皮巴拉   ┌──────────┐
│ 未注册用户│──────────→│ 新用户    │───────────────→│ 活跃用户  │
│         │           │ (无卡皮)  │                │          │
└─────────┘           └──────────┘                └────┬─────┘
                                                      │
                           ┌──────────────────────────┘
                           │
                           ▼
              ┌─────── 日常循环 ────────────┐
              │                            │
              │  聊天 → 分享记忆 → 旅行      │
              │    ↑                  │     │
              │    │    阅读手记 ←────┘     │
              │    │        │              │
              │    └────────┘              │
              │                            │
              └────────────────────────────┘
```

**关键节点：**

| 阶段 | 触发 | 页面 | 数据变化 |
|------|------|------|---------|
| 注册 | 用户提交表单 | /register | auth.users + profiles |
| 创建卡皮巴拉 | 首次进入 /chat | /chat (setup视图) | capybaras 新记录 |
| 首次对话 | 发送第一条消息 | /chat | conversations 首条 |
| 首次记忆 | 对话中 AI 提取记忆 | /chat | memories 首条 |
| 首次旅行 | 点击出发或对话触发 | /travel | travels 首条 |
| 首篇手记 | 旅行第一天晚间生成 | /journal | journals 首条 |
| 收获物品 | 手记中发现小物 | /journal | journals.item_found |

## 4.2 卡皮巴拉生命周期

```
创建
  ↓
[home] 在家休息
  ↓ ← 休息结束
  ├──→ 聊天 → 心情变化 + 记忆提取
  ├──→ 被触发旅行意愿
  │      ↓
  │    [traveling] 外出旅行（1-5天）
  │      ↓
  │    每日生成手记 + 发现小物
  │      ↓
  │    estimated_return 到达 → 惰性完成
  │      ↓
  │    [resting] 休息中（1-2天）
  │      ↓
  │    休息日生活层：
  │      sleep / swim / idle / eat / gaze
  │      ↓
  │    休息结束 → 返回 [home]
  │
  └──→ 被触发串门
         ↓
       [visiting]
         ↓
       返回 [home]
```

状态转换规则：

| 当前状态 | 事件 | 目标状态 |
|---------|------|---------|
| (无) | 用户创建 | home |
| home | 触发旅行 | traveling |
| home | 触发串门 | visiting |
| traveling | 旅行完成（惰性检查） | resting |
| resting | 休息结束 | home |
| visiting | 串门归来 | home |

## 4.3 旅行生命周期

```
                 idle (home)
                  │
     用户点击出发 / 对话触发
                  │
                  ▼
        ┌─────────────────┐
        │   创建旅行记录    │  POST /api/travel
        │  status: ongoing │
        │  capybara→traveling│
        │  duration: 1-5天  │
        │  destination: 真实地点│
        └────────┬────────┘
                 │
     ┌───── 旅行进行中 ──────┐
     │                       │
     │  每日生成手记           │
     │  有匹配时融入相遇叙事   │
     │  每日发现一件小物       │
     │                       │
     │  第1天 = 新鲜好奇      │
     │  中间  = 深入探索      │
     │  最后  = 不舍离开      │
     │                       │
     └──────────┬────────────┘
                │
                ▼
     estimated_return 时间到达
                │
                ▼
     用户打开页面 → GET /api/travel
                │
                ▼
        ┌─────────────────┐
        │   惰性完成检查    │
        │  status→completed │
        │  capybara→resting │
        │  completed_at=now │
        └────────┬────────┘
                 │
                 ▼
        ┌─────────────────┐
        │   休息阶段        │  1-2天
        │  status: resting │
        │  生活层动画：      │
        │  sleep/swim/idle │
        │  /eat/gaze       │
        └────────┬────────┘
                 │
                 ▼
          休息结束 → capybara→home → 回到 idle
```

## 4.4 对话会话生命周期

```
用户进入 /chat
  │
  ├─ 无卡皮巴拉 → 显示命名界面 → POST /api/capybara → 创建
  │
  └─ 有卡皮巴拉 → 加载最近 50 条消息 → 显示聊天界面
       │
       ├─ 用户发送消息
       │    ↓
       │  [乐观更新] 立即显示用户消息
       │    ↓
       │  POST /api/chat → AI 处理
       │    ↓
       │  显示 AI 回复 + 更新心情
       │    ↓
       │  AI 提取记忆 → 存储至 memories 表
       │    ↓
       │  显示 memory_reaction（卡皮的记忆意愿表达）
       │    ↓
       │  如果 want_to_travel → 显示旅行提示卡
       │
       └─ 每 15 秒检查旅行状态
            ↓
          如果旅行完成 → 显示"卡皮巴拉回来了"提示
```
