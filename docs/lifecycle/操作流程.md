## 5.2 旅行触发流程

```
触发方式 A：对话触发
  用户聊天 → AI 返回 want_to_travel: true
  → 前端显示旅行提示卡
  → 用户点击"让它去吧"
  → POST /api/travel

触发方式 B：直接出发
  用户进入 /travel → 看到出发按钮
  → 点击"让{名字}出发旅行"
  → POST /api/travel

触发条件检查：
  1. capybara.status === 'home'  （必须在家）
  2. 无 ongoing 旅行              （不能重复）

旅行参数：
  - duration: 1-5天（AI 根据记忆和意愿决定）
  - destination: 真实地点（AI 根据记忆共振选择）
```

## 5.3 页面导航流程

```
BottomNav 四个入口：
  💬 聊天 (/chat)  ←→  🗺️ 旅行 (/travel)  ←→  📖 手记 (/journal)  ←→  🏠 空间 (/home)

跳转逻辑：
  /chat    → 无条件可达（核心页面）
  /travel  → 无条件可达（根据状态显示不同视图）
  /journal → 无条件可达（展示手记时间线）
  /home    → 无条件可达（展示河岸空间 + 记忆库入口）

/travel 页面内部视图切换：
  有 ongoing 旅行  →  旅行进度视图 + 已生成手记
  无旅行          →  出发按钮 + 旅行历史

/journal 页面内部视图切换：
  有手记  →  手记时间线（按日展示）
  无手记  →  空态引导（"让卡皮去旅行吧"）
```

## 5.4 手记社交流程

```
手记 → 匹配相遇 → 逛对方河岸 → 留言 → 1v1 聊天

详细流程：
  [1] 用户收到每日手记推送（20:00 ± 30分钟）
       ↓
  [2] 阅读手记 → 发现匹配相遇段落
       ↓
  [3] 点击"去Ta的河岸看看 →"
       ↓
  [4] 进入对方河岸（脱敏展示）
       - 看到对方卡皮的收藏物品
       - 看到对方的 shareable 记忆主题（非原文）
       ↓
  [5] 留言互动
       - 在对方河岸留言板写一句话
       - 对方收到留言通知
       ↓
  [6] 双向留言后解锁 1v1 聊天
       - 通过卡皮转达的间接聊天
       - 保持主题级交流，不暴露隐私
```
