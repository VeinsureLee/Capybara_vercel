## 5.2 探索触发流程

```
触发方式 A：对话触发
  用户聊天 → AI 返回 want_to_explore: true
  → 前端显示探索提示卡
  → 用户点击"让它去吧"
  → POST /api/explore

触发方式 B：直接出发
  用户进入 /explore → 看到出发按钮
  → 点击"让{名字}出发探索"
  → POST /api/explore

触发条件检查：
  1. capybara.status === 'home'  （必须在家）
  2. 无 ongoing 探索              （不能重复）
```

## 5.3 页面导航流程

```
BottomNav 三个入口：
  💬 聊天 (/chat)   ←→   🗺️ 探索 (/explore)   ←→   🏠 空间 (/home)

跳转逻辑：
  /chat    → 无条件可达（核心页面）
  /explore → 无条件可达（根据状态显示不同视图）
  /home    → 无条件可达（展示河岸空间）

/explore 页面内部视图切换：
  有 ongoing 探索  →  ExploreScene 动画
  无探索          →  出发按钮 + 历史记录
```
