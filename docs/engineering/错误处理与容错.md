# 错误处理与容错

> API 错误处理、前端错误边界、AI 响应解析容错。

---

## 7.1 API 错误处理策略

| 场景 | 处理方式 | 用户感知 |
|------|---------|---------|
| AI 全部超时 | 使用本地模板兜底 | 正常回复（模板） |
| Supabase 不可达 | 返回 500 + 错误信息 | "加载失败，请重试" |
| 认证过期 | Middleware 刷新 token | 无感知 |
| 探索创建失败 | 回滚 capybara 状态 | "出发失败，请重试" |
| 重复创建探索 | 数据库检查 ongoing | "卡皮巴拉正在探索中" |

## 7.2 前端错误边界

```
页面级错误处理（建议添加）：
  app/error.tsx      → 全局错误页面
  app/not-found.tsx  → 404 页面
  app/loading.tsx    → 全局加载状态

组件级：
  try-catch 包裹 fetch 调用
  loading/error 状态变量
  用户友好的错误提示文案
```

## 7.3 AI 响应解析容错

```typescript
// 当前实现的解析逻辑
function parseAIResponse(raw: string): ChatResponse {
  try {
    // 尝试直接 JSON.parse
    return JSON.parse(raw)
  } catch {
    // 尝试提取 JSON 块
    const match = raw.match(/\{[\s\S]*\}/)
    if (match) return JSON.parse(match[0])

    // 完全兜底：把原文当 reply
    return {
      reply: raw,
      mood: 'calm',
      keywords: [],
      want_to_explore: false
    }
  }
}
```
