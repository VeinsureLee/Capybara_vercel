# 性能设计

> 关键指标、优化策略、AI 延迟优化。

---

## 9.1 关键指标

| 指标 | 目标 | 当前 |
|------|------|------|
| 首屏加载 (FCP) | ≤ 1.5s | ~1.2s (Vercel CDN) |
| 对话响应延迟 | ≤ 5s (P95) | 2-25s (取决于 AI) |
| 探索页动画帧率 | 60fps | 60fps (CSS only) |
| 数据库查询 | ≤ 100ms | ~50ms |
| 包大小 (JS) | ≤ 200KB gzip | ~150KB |

---

## 9.2 优化策略

| 策略 | 说明 |
|------|------|
| 代码分割 | Next.js 自动按路由分割 |
| CSS 动画 | 零 JS 动画库，GPU 加速 transform/opacity |
| 图片零依赖 | 全部使用 emoji，无图片资源加载 |
| 乐观更新 | 发送消息时立即显示，不等 API 返回 |
| 惰性完成 | 无 Cron Job，客户端触发完成检查 |
| 数据库索引 | conversations(user_id, created_at), explorations(user_id, status) |

---

## 9.3 AI 延迟优化

```
当前：串行调用降级链，最差 75s（3×25s 超时）
优化方案：
  1. 降低单次超时至 15s
  2. 缓存最近 AI 配置状态，跳过已知不可用的 provider
  3. 未来考虑 streaming response（SSE）减少感知延迟
```
