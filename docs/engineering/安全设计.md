# 安全设计

> 认证安全、数据安全、输入验证。

---

## 8.1 认证安全

| 措施 | 实现 |
|------|------|
| Session Cookie | httpOnly + Secure + SameSite=Lax |
| Token 刷新 | Middleware 每次请求自动刷新 |
| 密码存储 | Supabase Auth 管理（bcrypt） |
| API 鉴权 | 每个 Route 调用 `getUser()` 验证 |

---

## 8.2 数据安全

| 措施 | 实现 |
|------|------|
| RLS | 所有表启用行级安全策略 |
| SQL 注入 | Supabase SDK 参数化查询 |
| XSS | React 自动转义 + Next.js CSP |
| API Key | 服务端环境变量，不暴露给前端 |
| CORS | Next.js 默认同源策略 |

---

## 8.3 输入验证

```
用户消息：
  - 最大长度：500 字符（建议添加）
  - 内容审核：MVP 暂无，未来接入审核 API
  - Rate Limit：建议每分钟 20 条

卡皮巴拉命名：
  - 最大长度：20 字符
  - 不允许空名
  - 不允许特殊字符（建议添加）
```
