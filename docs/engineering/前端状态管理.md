# 前端状态管理

> 当前方案与未来优化方向。

---

## 6.1 当前方案：React Local State

每个页面独立管理状态，通过 API 调用同步：

```
/chat 页面状态：
  capybara: Capybara | null        // 卡皮巴拉信息
  messages: Message[]               // 消息列表
  input: string                     // 输入框
  loading: boolean                  // 加载状态
  sending: boolean                  // 发送中
  explorationStatus: string | null  // 探索状态

/explore 页面状态：
  capybara: Capybara | null
  currentExploration: Exploration | null
  pastExplorations: Exploration[]
  loading: boolean

/home 页面状态：
  capybara: Capybara | null
  items: ExplorationItem[]
  loading: boolean

/travel 页面状态 (V2)：
  capybara: Capybara | null
  currentTravel: Travel | null
  travelHistory: Travel[]
  matchedEncounter: Encounter | null
  loading: boolean

/journal 页面状态 (V2)：
  journals: Journal[]
  currentJournal: Journal | null
  encounterSegments: EncounterSegment[]
  loading: boolean

/memory 页面状态 (V2, 卡皮的小本子)：
  memories: Memory[]
  filter: 'all' | 'shareable' | 'private'
  editingMemory: Memory | null
  loading: boolean
```

## 6.2 未来优化方向

当跨页面状态同步需求增加时，考虑引入：

| 方案 | 适用场景 | 复杂度 |
|------|---------|-------|
| React Context | 共享 capybara/auth 状态 | 低 |
| Zustand | 全局状态管理 | 中 |
| SWR/React Query | API 缓存 + 自动重新获取 | 中 |
| Supabase Realtime | 实时状态同步 | 高 |

**推荐 MVP+1 优化路径：**
1. 提取 `useCapybara()` 自定义 Hook + React Context
2. 引入 SWR 做 API 缓存和乐观更新
3. Supabase Realtime 订阅 explorations 表变更

### V2 新增优化需求

| 页面/功能 | 优化策略 | 说明 |
|-----------|---------|------|
| /memory 记忆小本子 | 离线缓存（Service Worker + IndexedDB） | 用户频繁查阅记忆，需支持离线浏览已加载的记忆列表 |
| /journal 手记列表 | 分页 + 无限滚动（Intersection Observer） | 手记随时间积累量大，首屏加载最近 10 条，滚动触底加载更多 |
| /travel 旅行状态 | 定时轮询（每 5 分钟） | 多日旅行期间需定期检查状态变更（偶遇事件、旅行完成），使用 `setInterval` + `visibilitychange` 优化（页面不可见时暂停轮询） |
| 记忆可见性切换 | 乐观更新（Optimistic Update） | 用户切换 shareable/private 时立即更新 UI，后台异步同步到服务端，失败时回滚并提示 |

## 6.3 V2 跨页面状态

V2 引入多个需要跨页面同步的状态，建议通过 React Context 或 Zustand 统一管理：

### 旅行状态影响聊天人设

```
TravelContext:
  travelStatus: 'home' | 'traveling' | 'resting'
  currentTravel: Travel | null
  daysRemaining: number | null
```

卡皮巴拉的旅行状态（home / traveling / resting）直接影响 /chat 页面的对话人设：
- `home`：正常懒洋洋聊天模式
- `traveling`：旅途中回复带有旅行见闻，回复频率降低（模拟信号不好）
- `resting`：刚旅行回来，分享旅途感悟，偶尔犯困

Chat 页面需订阅 travelStatus 变化，动态切换 prompt 模板。

### 记忆提取影响聊天 UI

```
MemoryContext:
  lastExtractedMemory: Memory | null
  memoryReaction: string | null    // AI 对新记忆的反应文案
  showMemoryToast: boolean
```

当 AI 从对话中提取到新记忆时：
- Chat 页面显示 `memory_reaction` 气泡（"卡皮把这件事记在小本子里了"）
- 底部导航 /memory 图标显示小红点
- 提取结果通过 Context 广播，memory 页面自动刷新列表

### 手记通知徽章

```
JournalContext:
  unreadJournalCount: number
  latestJournalId: string | null
```

每晚生成新手记时：
- 底部导航 /journal 图标显示未读数量徽章
- 用户进入 /journal 页面后清除未读计数
- 未读状态持久化到 localStorage，防止刷新丢失
