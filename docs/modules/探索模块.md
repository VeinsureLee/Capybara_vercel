## 关键词聚合算法

从最近 20 条对话中提取关键词，按时间衰减加权，取 top 8 作为探索主题输入。

```typescript
// 最近 20 条对话的关键词，按时间衰减加权
function aggregateKeywords(conversations: Conversation[]): string[] {
  const weightMap = new Map<string, number>()

  conversations.forEach((conv, index) => {
    const weight = Math.pow(0.85, index)  // 指数衰减：越近权重越大
    ;(conv.keywords || []).forEach(kw => {
      weightMap.set(kw, (weightMap.get(kw) || 0) + weight)
    })
  })

  return [...weightMap.entries()]
    .sort((a, b) => b[1] - a[1])
    .slice(0, 8)                          // 取 top 8 关键词
    .map(([kw]) => kw)
}
```

算法特点：
- 指数衰减系数 0.85，最近的对话权重最高
- 同一关键词在多条对话中出现时权重累加
- 最终取权重最高的 8 个关键词

## 惰性完成机制

探索完成不依赖定时任务或 Cron Job，而是在用户查询时惰性检查并完成。

```
GET /api/explore 时：
  1. 查询 status='ongoing' 的探索
  2. 如果 estimated_return < now：
     - 更新 exploration.status = 'completed'
     - 更新 capybara.status = 'home'
     - 返回 { exploration, just_completed: true }
  3. 如果还在进行中：
     - 返回 { exploration, just_completed: false }
  4. 如果无进行中探索：
     - 返回 { exploration: null }
```

优势：零运维成本，无需后台定时任务。

## 探索触发方式

### 方式 A：对话触发

```
用户聊天 → AI 返回 want_to_explore: true
  → 前端显示探索提示卡
  → 用户点击"让它去吧"
  → POST /api/explore
```

### 方式 B：直接出发

```
用户进入 /explore → 看到出发按钮
  → 点击"让{名字}出发探索"
  → POST /api/explore
```

### 触发条件检查

1. `capybara.status === 'home'` （必须在家）
2. 无 ongoing 探索（不能重复）

## 探索生命周期

```
                 idle
                  │
     用户点击出发 / 对话触发
                  │
                  ▼
        ┌─────────────────┐
        │   创建探索记录    │  POST /api/explore
        │  status: ongoing │
        │  capybara→exploring│
        └────────┬────────┘
                 │
     ┌───────── 时间流逝 ──────────┐
     │                             │
     │  前端每秒更新进度条           │
     │  progress = (now-start)/(end-start) │
     │                             │
     │  0-25%  → DepartureStage    │
     │  25-75% → JourneyStage     │
     │  75-95% → DiscoveryStage   │
     │  95%+   → ReturnStage      │
     │                             │
     └────────────┬────────────────┘
                  │
                  ▼
     estimated_return 时间到达
                  │
                  ▼
     用户打开页面 → GET /api/explore
                  │
                  ▼
        ┌─────────────────┐
        │   惰性完成检查    │
        │  status→completed │
        │  capybara→home   │
        │  completed_at=now │
        └────────┬────────┘
                 │
                 ▼
            归来动画
                 │
                 ▼
          用户确认收获 → 回到 idle
```
