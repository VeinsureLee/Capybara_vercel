# M3: 对话模块 (`chat`)

> 处理用户与卡皮巴拉之间的消息交互，包含 AI 推理、情绪分析、关键词提取和探索意愿判断。

---

## 模块概览

| 属性 | 说明 |
|------|------|
| 职责 | 用户消息收发、AI 推理、情绪分析、关键词提取、探索意愿判断 |
| 入口 | `/api/chat` (POST), `/chat` 页面 |
| 依赖 | AI Client, Supabase, Prompts |
| 性能 | 单次调用 ≤ 25s（AI 超时），P95 目标 ≤ 5s |

---

## 消息处理管线

```
用户消息
  ↓
[1] 鉴权 → 获取 user_id
  ↓
[2] 查询 capybara（含 status, mood, memory, traits）
  ↓
[3] 查询最近 20 条 conversations
  ↓
[4] 构建 System Prompt
     - 注入卡皮巴拉名字、性格、心情、状态
     - 注入长期记忆 top keywords
     - 如果 status=exploring：加入探索上下文
  ↓
[5] 构建 User Prompt
     - 最近 10 条对话历史
     - 当前用户消息
  ↓
[6] 调用 AI（DeepSeek → Claude → OpenAI → fallback）
  ↓
[7] 解析 JSON 响应
     {
       reply: "回复文本",
       mood: "happy",
       keywords: ["关键词1", "关键词2"],
       want_to_explore: false
     }
  ↓
[8] 存储对话记录（user + capybara 两条）
  ↓
[9] 更新 capybara.mood
  ↓
[10] 追加 keywords 到 capybara.memory
  ↓
[11] 返回响应给前端
```

---

## Prompt 差异化策略

根据卡皮巴拉当前状态，System Prompt 会注入不同的上下文：

| 卡皮巴拉状态 | System Prompt 特征 |
|-------------|-------------------|
| home | "你在河岸休息，可以跟主人聊天，可以提出想去探索" |
| exploring | "你正在外面探索，可以分享见闻" |
| visiting | "你在别人的河岸串门" |

## AI 响应解析容错

```typescript
function parseAIResponse(raw: string): ChatResponse {
  try {
    // 尝试直接 JSON.parse
    return JSON.parse(raw)
  } catch {
    // 尝试提取 JSON 块
    const match = raw.match(/\{[\s\S]*\}/)
    if (match) return JSON.parse(match[0])

    // 完全兜底：把原文当 reply
    return {
      reply: raw,
      mood: 'calm',
      keywords: [],
      want_to_explore: false
    }
  }
}
```

## 前端状态管理

```
/chat 页面状态：
  capybara: Capybara | null        // 卡皮巴拉信息
  messages: Message[]               // 消息列表
  input: string                     // 输入框
  loading: boolean                  // 加载状态
  sending: boolean                  // 发送中
  explorationStatus: string | null  // 探索状态
```

交互流程：
- 用户发送消息时执行乐观更新，立即显示用户消息
- 等待 AI 回复后显示卡皮巴拉消息 + 更新心情
- 如果 AI 返回 `want_to_explore: true`，显示探索提示卡
- 每 15 秒检查探索状态，探索完成时显示"卡皮巴拉回来了"提示
