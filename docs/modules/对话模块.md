# M3: 对话模块 (`chat`)

> 处理用户与卡皮巴拉之间的消息交互，包含 AI 推理、情绪分析、记忆提取、旅行意愿判断和记忆反应表达。

---

## 模块概览

| 属性 | 说明 |
|------|------|
| 职责 | 用户消息收发、AI 推理、情绪分析、记忆提取、旅行意愿判断、记忆反应 |
| 入口 | `/api/chat` (POST), `/chat` 页面 |
| 依赖 | AI Client, Supabase, Prompts |
| 性能 | 单次调用 ≤ 25s（AI 超时），P95 目标 ≤ 5s |

---

## 消息处理管线（V2）

```
用户消息
  ↓
[1] 鉴权 → 获取 user_id
  ↓
[2] 查询 capybara（含 status, mood, equipped_costumes, current_life_action）
  ↓
[3] 查询最近 20 条 conversations
  ↓
[4] 查询相关记忆（memories 表，按主题相关性）
  ↓
[5] 构建 System Prompt
     - 注入卡皮巴拉名字、心情、状态
     - 注入对话历史（AI 读历史理解卡皮性情，不注入结构化性格）
     - 注入相关记忆上下文
     - 根据 status 差异化注入上下文（见下方 Prompt 差异化策略）
  ↓
[6] 构建 User Prompt
     - 最近 10 条对话历史
     - 当前用户消息
  ↓
[7] 调用 AI（DeepSeek → Claude → OpenAI → fallback）
  ↓
[8] 解析 JSON 响应
     {
       reply: "回复文本",
       mood: "happy",
       keywords: ["关键词1", "关键词2"],
       want_to_travel: false,
       memory_reaction: null | { content: "...", shareable: true }
     }
  ↓
[9] 存储对话记录（user + capybara 两条）
  ↓
[10] 更新 capybara.mood
  ↓
[11] 记忆提取（V2 新增）
     - 如果 memory_reaction 非空 → 写入 memories 表
     - AI 自动分类 shareable / private
     - 敏感话题标记 sensitive=true
  ↓
[12] 追加 keywords 到 capybara.memory（V1 Legacy，保留向后兼容）
  ↓
[13] 返回响应给前端
```

## Prompt 差异化策略（V2）

根据卡皮巴拉当前状态，System Prompt 注入不同的上下文：

| 卡皮巴拉状态 | System Prompt 特征 |
|-------------|-------------------|
| home | "你在河岸休息，可以跟主人聊天，可以提出想去旅行" |
| traveling | "你正在{地点名}旅行（第{N}天），可以分享当地见闻，回复内嵌当下地点情境" |
| resting | "你刚旅行回来在休息，可以聊旅途见闻和归来感受，也可以回忆旅途中遇到的事" |

### traveling 状态的地点上下文

当 `status=traveling` 时，System Prompt 额外注入：

```
- 地点名（如"京都·下鸭神社附近"）
- 第几天（如"旅行第3天"）
- 地点描述标签（如"樱花、古寺、石板路"）
- 当日已发生的手记概要（如有）
```

这让卡皮的回复自然体现"在路上"的时空感。

### V1 Legacy Prompt 状态（保留向后兼容）

| V1 状态 | V1 System Prompt 特征 |
|---------|----------------------|
| exploring | "你正在外面探索，可以分享见闻" |
| visiting | "你在别人的河岸串门" |

## AI 响应格式（V2）

```json
{
  "reply": "回复文本",
  "mood": "happy",
  "keywords": ["关键词1", "关键词2"],
  "want_to_travel": false,
  "memory_reaction": null
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| reply | string | 卡皮的回复文本 |
| mood | string | 当前心情（happy/curious/sleepy/excited/calm） |
| keywords | string[] | 对话关键词（V1 兼容） |
| want_to_travel | boolean | 是否想出发旅行（替代 V1 的 want_to_explore） |
| memory_reaction | object \| null | 记忆反应（V2 新增） |

### memory_reaction 结构

当 AI 识别到值得记住的内容时，返回 memory_reaction：

```json
{
  "content": "用户分享的记忆摘要",
  "topic": "主题标签",
  "shareable": true,
  "expression": "这件事挺有意思的，我想记着"
}
```

卡皮的记忆反应表达：

| shareable | 卡皮表达示例 |
|-----------|------------|
| true | "这件事挺有意思的，我想记着" / "下次出去我可以跟别的卡皮聊聊这个" |
| false | "这个就咱俩知道" / "我不跟别人说" |

## 记忆提取流程（V2 新增）

```
对话完成，AI 返回 memory_reaction
  ↓
memory_reaction 非空？
  ├── 否 → 跳过
  └── 是 ↓
        检查敏感话题（自杀/自伤/暴力/未成年人等）
          ├── 敏感 → 写入 memories 表，shareable=false, sensitive=true
          └── 非敏感 → 写入 memories 表，shareable 取 AI 判定值
                ↓
              前端展示卡皮的记忆表达（expression 字段）
```

记忆提取是被动的——AI 在对话中自然判断哪些内容值得记住，不是每条对话都触发。用户可在"卡皮记忆库"查看和管理所有记忆。

## AI 响应解析容错

```typescript
function parseAIResponse(raw: string): ChatResponse {
  try {
    // 尝试直接 JSON.parse
    return JSON.parse(raw)
  } catch {
    // 尝试提取 JSON 块
    const match = raw.match(/\{[\s\S]*\}/)
    if (match) return JSON.parse(match[0])

    // 完全兜底：把原文当 reply
    return {
      reply: raw,
      mood: 'calm',
      keywords: [],
      want_to_travel: false,
      memory_reaction: null
    }
  }
}
```

## 前端状态管理（V2）

```
/chat 页面状态：
  capybara: Capybara | null        // 卡皮巴拉信息
  messages: Message[]               // 消息列表
  input: string                     // 输入框
  loading: boolean                  // 加载状态
  sending: boolean                  // 发送中
  travelStatus: {                   // V2: 旅行状态（替代 explorationStatus）
    status: string | null           // traveling / resting / null
    location: string | null         // 当前旅行地点名
    day: number | null              // 旅行第几天
  }
  memoryReaction: {                 // V2: 记忆反应
    expression: string | null       // 卡皮的记忆表达文本
    shareable: boolean | null       // 可分享/私密
  } | null
  explorationStatus: string | null  // V1 Legacy
```

交互流程：
- 用户发送消息时执行乐观更新，立即显示用户消息
- 等待 AI 回复后显示卡皮巴拉消息 + 更新心情
- 如果 AI 返回 `want_to_travel: true`，显示旅行提示卡（替代 V1 的探索提示卡）
- 如果 AI 返回 `memory_reaction`，在消息气泡下方展示卡皮的记忆表达（如"这个我想记着"）
- 每 15 秒检查旅行状态，旅行完成时显示"卡皮巴拉回来了"提示
- V1 Legacy：每 15 秒检查探索状态，探索完成时显示提示
