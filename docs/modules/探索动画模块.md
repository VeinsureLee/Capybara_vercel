# M5: 探索动画模块 (`explore-animation`)

> 将卡皮巴拉的探索过程可视化为四阶段动画，使用纯 CSS/Tailwind 动画和 React State 实现 60fps 沉浸式体验。

> **V2 架构变更说明：** V1 使用分钟级探索（3-12 分钟），通过实时进度计算驱动四阶段动画（出发→途中→发现→归来）。V2 改为 1-5 天的多日旅行模型，秒级实时动画不再适用于天级时间跨度。V2 保留简短的出发/归来过渡动画，核心可视化转为旅行状态卡片和每日手记展示。以下 V1 内容保留作为设计参考。

---

## V2: 旅行可视化

V2 的旅行持续 1-5 天，不再使用实时进度驱动的连续动画。可视化模型转变为状态卡片 + 事件驱动的短动画。

### 设计原则

- **无实时进度动画**：1-5 天的旅行无法用秒级进度条表达，放弃连续动画
- **状态卡片为核心**：用户打开 App 时看到旅行状态卡片，而非动画场景
- **短动画仅用于关键时刻**：出发、归来、手记揭示等离散事件

### 旅行状态卡片

旅行期间，主界面显示旅行状态卡片，替代 V1 的全屏动画场景。

```
┌─────────────────────────────────────┐
│  ✈️ {name} 正在旅行中                │
│                                     │
│  📍 目的地：{location.name}          │
│  🌍 {location.country} · {region}   │
│                                     │
│  ██████████████░░░░░░░░░░  58%     │
│  ─┬──────┬──────┬──────┬─          │
│  出发   第1天   第2天   到达         │
│                ↑ 现在               │
│                                     │
│  📅 第 2 / 3 天 · 还有 1 天 4 小时   │
│                                     │
│  💬 最近消息："在海边捡到一颗贝壳..."  │
│                                     │
│  ┌─────────────────────────────────┐│
│  │  📖 生成今日手记                 ││
│  └─────────────────────────────────┘│
└─────────────────────────────────────┘
```

卡片信息：
- 目的地名称、国家、地区
- **连续进度条**（V2.1 改进）：基于时间的百分比进度，每秒刷新，天数分隔线作为刻度标注
- 旅行进度：第 X 天 / 共 N 天 + 剩余时间倒计时
- 最新一条旅途消息摘要
- 手记生成按钮（带阻塞机制：生成中置灰+loading，防止重复点击）

### 出发动画（5 秒）

简短的告别过渡动画，在用户确认出发后播放。

```
时间线：0s ──────── 2s ──────── 4s ── 5s
        │           │           │     │
        │ 卡皮巴拉   │ 背上行李   │ 走出 │→ 切换到状态卡片
        │ 站在家门口  │ 挥手告别   │ 画面 │
```

| 元素 | 动画 | 时长 |
|------|------|------|
| 卡皮巴拉 | 原地 → 背上行李 → 向右走出画面 | 5s |
| 告别文字 | "出发去{location}啦~ 过几天回来！" 淡入淡出 | 2s |
| 背景 | 家园场景 → 渐隐 | 最后 1s |

### 归来动画（5 秒）

旅行完成后的欢迎回家过渡动画。

```
时间线：0s ──────── 2s ──────── 4s ── 5s
        │           │           │     │
        │ 从左侧     │ 走到中央   │ 展示 │→ 切换到纪念品/手记
        │ 走入画面    │ 放下行李   │ 纪念品│
```

| 元素 | 动画 | 时长 |
|------|------|------|
| 卡皮巴拉 | 从左侧走入 → 到中央停下 | 3s |
| 纪念品揭示 | 行李中弹出 1-3 件纪念品 + 闪光 | 2s |
| 欢迎文字 | "欢迎回来！带了好东西~" 弹入 | 1s |
| 撒花 | 轻量撒花效果 | 2s |

### 休息状态动画

旅行归来后卡皮巴拉进入 resting 状态，主界面显示休息场景。

```
┌─────────────────────────────────────┐
│                                     │
│        💤                           │
│     ～～🦫～～                        │
│    （卡皮巴拉蜷缩在垫子上）           │
│                                     │
│  {name} 旅行回来了，正在休息...       │
│  "嗯...好累...让我躺一会儿..."       │
│                                     │
└─────────────────────────────────────┘
```

| 元素 | 动画 | CSS 实现 |
|------|------|---------|
| 卡皮巴拉 | 缓慢呼吸起伏 | `scale(1↔1.03) infinite 3s ease-in-out` |
| 💤 气泡 | 周期性浮出消散 | `translateY + opacity + scale, infinite 4s` |
| 背景 | 暖色调柔光 | `background: warm gradient, subtle pulse` |

## V2: 手记展示动画

每日手记（journal）以 Webtoon 风格的垂直滚动叙事呈现，是 V2 核心视觉体验。

### Webtoon 垂直滚动

手记内容以面板（panel）序列垂直排列，用户向下滚动阅读。

```
┌─────────────────────────────────┐
│  📖 第 2 天的手记                │
│  {location.name} · {date}       │
├─────────────────────────────────┤
│                                 │
│  ┌─────────────────────────┐    │
│  │ [场景面板]               │    │  ← scroll-reveal: 滚动到视口时淡入
│  │ "清晨，镰仓的海风带着    │    │
│  │  咸味吹过来..."          │    │
│  └─────────────────────────┘    │
│             ↓                   │
│  ┌─────────────────────────┐    │
│  │ [对话面板]               │    │  ← slide-in: 从侧面滑入
│  │  🦫 "这个贝壳好漂亮"     │    │
│  └─────────────────────────┘    │
│             ↓                   │
│  ┌─────────────────────────┐    │
│  │ [旁白面板]               │    │  ← fade-in-up: 从下方淡入
│  │ 卡皮巴拉把贝壳小心地      │    │
│  │ 放进了包里。              │    │
│  └─────────────────────────┘    │
│             ↓                   │
│  ┌─────────────────────────┐    │
│  │ [傍晚面板]               │    │  ← slow-reveal: 缓慢展开
│  │ "夕阳把海面染成了         │    │
│  │  金色...今天也是          │    │
│  │  悠闲的一天呢"           │    │
│  └─────────────────────────┘    │
│                                 │
└─────────────────────────────────┘
```

### 面板过渡动画

每种面板类型有不同的入场动画，通过 Intersection Observer 在滚动到视口时触发。

| 面板类型 | 入场动画 | CSS 实现 |
|---------|---------|---------|
| scene（场景） | 滚动揭示：从模糊到清晰 | `filter: blur(8px→0) + opacity 0→1, 0.8s` |
| dialogue（对话） | 侧滑入：从左/右交替滑入 | `translateX(±30px→0) + opacity, 0.5s` |
| narration（旁白） | 下方淡入 | `translateY(20px→0) + opacity, 0.6s` |
| 结尾面板 | 缓慢展开 | `max-height 0→auto + opacity, 1.2s` |

### 偶遇片段高亮

当手记包含偶遇片段（encounter_segment）时，以特殊视觉效果呈现两只卡皮巴拉的相遇。

```
            ↓ 正常面板区域 ↓
  ┌─────────────────────────┐
  │ [场景面板]               │
  │ "在海边散步时..."        │
  └─────────────────────────┘
            ↓
  ╔═══════════════════════════╗  ← 偶遇区域：金色边框 + 柔光背景
  ║  ✨ 偶遇                  ║
  ║                           ║
  ║  🦫 ←→ 🦫                ║  ← 两只卡皮巴拉 glow 动画
  ║                           ║
  ║  "咦，那边也有一只         ║
  ║   卡皮巴拉在散步呢..."     ║
  ║                           ║
  ║  它们在沙滩上一起          ║
  ║  晒了一会儿太阳。          ║
  ╚═══════════════════════════╝
            ↓
  ┌─────────────────────────┐
  │ [后续面板]               │
  └─────────────────────────┘
```

| 元素 | 动画 | CSS 实现 |
|------|------|---------|
| 偶遇区域边框 | 金色脉冲光效 | `box-shadow: 0 0 20px gold, pulse infinite 2s` |
| 偶遇背景 | 渐变柔光 | `background: linear-gradient(warm-gold), opacity pulse` |
| 两只卡皮巴拉 | 靠近 + 发光 | `translateX(±50px→0) + filter: drop-shadow(glow)` |
| ✨ 星星 | 散布闪烁 | `scale pulse + opacity flicker, random delay` |


# 探索动画设计方案（V1, 保留参考）

> 将卡皮巴拉的探索过程拆分为四个动画阶段，每个阶段有独立场景、动画效果和交互

---

## 一、设计目标

当前探索页面只有一个倒计时和静态文字，缺乏沉浸感。本方案将探索过程可视化为四个动画阶段，让用户能"看到"卡皮巴拉的旅途，增强情感投入和期待感。

### 核心原则

- **纯 CSS/Tailwind 动画**，不引入额外动画库
- **Emoji 为主要视觉元素**，与现有风格一致
- **根据探索进度自动推进阶段**，无需用户操作
- **每个阶段有独立的背景、动画和文字**

---

## 二、四阶段设计

### 阶段流转

```
时间进度：0%                25%                75%                95%          100%
         ├─── 🎒 出发 ───┼─── 🚶 途中探索 ───┼─── ✨ 发现物品 ───┼─── 🏠 归来 ─┤
```

进度计算公式：
```
progress = (当前时间 - 出发时间) / (预计返回时间 - 出发时间) × 100%
```

### 阶段 1：出发（0% - 25%）

**场景描述：** 河岸背景，卡皮巴拉背上小包，向右走出画面

**视觉元素：**
- 背景：渐变绿色河岸 + 水波纹
- 前景：草丛、小花（静态装饰）
- 卡皮巴拉：🦫 从画面中央向右移动

**动画效果：**
| 元素 | 动画 | CSS 实现 |
|------|------|---------|
| 卡皮巴拉 | 上下轻微弹跳 + 从中间移到右侧 | `translateX(0→100%) + translateY bounce` |
| 背包 emoji 🎒 | 跟随卡皮巴拉 | 相对定位 |
| 挥手文字 | 淡入显示 "出发啦~ 去看看有什么好玩的！" | `opacity 0→1, fadeIn` |
| 背景云朵 ☁️ | 缓慢从右向左飘 | `translateX infinite linear` |

**文案：**
- "🎒 {name}背起小包出发啦~"
- "目的地：{location.name}"
- "{location.description}"

---

### 阶段 2：途中探索（25% - 75%）

**场景描述：** 场景根据探索主题变化（自然/神秘/怀旧/梦幻/冒险），卡皮巴拉在画面中走路

**视觉元素 - 按主题切换：**

| 主题 | 背景色 | 飘过的元素 | 地面元素 |
|------|--------|-----------|---------|
| 自然 | 绿→蓝渐变 | ☁️ 🦋 🍃 | 🌿 🌺 🌼 |
| 神秘 | 紫→深蓝渐变 | ✨ 🌙 💫 | 🍄 🌑 🔮 |
| 怀旧 | 暖黄→橙渐变 | ☁️ 🍂 🕊️ | 🏮 🌾 🪨 |
| 梦幻 | 粉→淡紫渐变 | ⭐ 🌈 💭 | 🌸 🦄 🎀 |
| 冒险 | 灰→深绿渐变 | 🦅 ☁️ 💨 | ⛰️ 🪨 🌲 |

**动画效果：**
| 元素 | 动画 | CSS 实现 |
|------|------|---------|
| 卡皮巴拉 🦫 | 原地踏步（上下弹跳）| `translateY(-8px→0) infinite 0.6s` |
| 飘过元素 | 从右向左飘过，随机高度和速度 | `translateX(100vw→-50px) infinite, 不同 duration` |
| 地面元素 | 从右向左匀速滚动 | `translateX infinite linear` |
| 途中见闻文字 | 周期性切换，淡入淡出 | `opacity transition, setInterval 切换` |

**途中见闻文案（从探索 story 中截取或模板）：**
- "在路上看到了一条清澈的小溪~"
- "这里好安静呀，只有风吹树叶的声音"
- "闻到了花香，好像离目的地不远了..."
- "遇到了一只小蝴蝶，它在带路呢！"

---

### 阶段 3：发现物品（75% - 95%）

**场景描述：** 卡皮巴拉到达目的地，停下来，物品逐个从地面弹出

**视觉元素：**
- 背景：目的地场景（静态）
- 卡皮巴拉：原地，做出惊喜表情（加 ❗ 或 ✨）
- 物品卡片：从底部弹出，带发光效果

**动画效果：**
| 元素 | 动画 | CSS 实现 |
|------|------|---------|
| 卡皮巴拉 | 停下 + 跳一下表示惊喜 | `scale(1→1.2→1) + translateY bounce once` |
| "!" 惊叹号 | 卡皮巴拉头顶弹出 | `scale(0→1) + fadeIn` |
| 物品卡片 | 从下方弹入 + 轻微弹跳 | `translateY(100%→0) + overshoot ease` |
| 闪光特效 ✨ | 物品周围的星星闪烁 | `scale pulse + opacity flicker infinite` |
| 稀有度光圈 | 根据稀有度显示不同颜色光圈 | `box-shadow glow animation` |

**稀有度视觉差异：**
| 稀有度 | 光圈颜色 | 弹出延迟 | 额外效果 |
|--------|---------|---------|---------|
| common | 无光圈 | 0ms | 无 |
| uncommon | 绿色柔光 | 200ms | 轻微闪烁 |
| rare | 蓝色光圈 | 400ms | 持续闪烁 + 放大 |
| legendary | 紫金色光圈 | 600ms | 全屏闪光 + 慢动作弹出 |

---

### 阶段 4：归来（95% - 100%）

**场景描述：** 回到河岸家园，卡皮巴拉从左侧走入，庆祝动画

**视觉元素：**
- 背景：用户的河岸场景
- 卡皮巴拉：从左走入画面中央
- 庆祝特效：撒花 / 星星

**动画效果：**
| 元素 | 动画 | CSS 实现 |
|------|------|---------|
| 卡皮巴拉 | 从左走入 + 到中间停下 | `translateX(-100%→0)` |
| 撒花 🎊 | 多个 emoji 从上方飘落 | `translateY(-100vh→100vh) + rotate, random delay` |
| "欢迎回来" 文字 | 弹入 | `scale(0→1.1→1)` |
| 收获总结卡片 | 从底部滑入 | `translateY(100%→0)` |

**文案：**
- "🎉 {name}回来了！"
- "这次探索带回了 {items.length} 件东西~"

## 三、进度条设计

页面底部始终显示探索进度条：

```
🎒 出发        🚶 途中         ✨ 发现        🏠 归来
●━━━━━━━━━━━━●━━━━━━━━━━━━━●━━━━━━━━━━━○
                            ↑ 当前位置 (78%)
```

- 使用四个节点标记阶段
- 已过阶段显示实心圆 ●
- 当前阶段闪烁
- 未到阶段显示空心圆 ○
- 进度条颜色随阶段变化（绿 → 蓝 → 金 → 绿）

## 四、文件结构

```
src/
├── app/explore/page.tsx                    # 探索主页面（改造）
├── components/explore/
│   ├── ExploreScene.tsx                    # 场景容器（管理阶段切换）
│   ├── DepartureStage.tsx                  # 阶段1：出发
│   ├── JourneyStage.tsx                    # 阶段2：途中（含主题切换）
│   ├── DiscoveryStage.tsx                  # 阶段3：发现物品
│   ├── ReturnStage.tsx                     # 阶段4：归来
│   ├── ProgressBar.tsx                     # 底部进度条
│   └── FloatingElements.tsx                # 通用飘浮元素组件
└── app/globals.css                         # 新增动画 keyframes
```

### 组件职责

| 组件 | 职责 |
|------|------|
| `ExploreScene` | 接收探索数据，计算当前进度和阶段，渲染对应的 Stage 组件 |
| `DepartureStage` | 渲染出发场景 + 卡皮巴拉走出动画 |
| `JourneyStage` | 根据探索主题渲染途中场景 + 飘浮物 + 走路动画 + 循环见闻文字 |
| `DiscoveryStage` | 渲染发现场景 + 物品逐个弹出动画 |
| `ReturnStage` | 渲染归来场景 + 庆祝动画 + 收获总结 |
| `ProgressBar` | 四节点进度条 + 当前位置指示 |
| `FloatingElements` | 可复用的飘浮 emoji 动画（云朵、树叶、星星等） |

### 数据流

```
explore/page.tsx
  │
  ├─ GET /api/explore → 获取当前探索数据
  │   { status, started_at, estimated_return, story, items_found, trigger_keywords }
  │
  ├─ 计算 progress = (now - started) / (return - started)
  │
  ├─ 确定 stage:
  │   progress < 0.25  → 'departure'
  │   progress < 0.75  → 'journey'
  │   progress < 0.95  → 'discovery'
  │   progress >= 0.95 → 'return'
  │
  └─ <ExploreScene exploration={data} progress={progress} stage={stage} />
       ├─ stage === 'departure'  → <DepartureStage />
       ├─ stage === 'journey'    → <JourneyStage theme={location.theme} />
       ├─ stage === 'discovery'  → <DiscoveryStage items={items_found} />
       └─ stage === 'return'     → <ReturnStage items={items_found} />
       └─ <ProgressBar progress={progress} stage={stage} />
```

## 六、探索时间说明

为了方便测试和演示，MVP 阶段的探索时间较短：

| 类型 | 实际时长 | 各阶段时长 |
|------|---------|-----------|

| medium | 6 分钟 | 出发 90s → 途中 180s → 发现 72s → 归来 18s |
| long | 12 分钟 | 出发 180s → 途中 360s → 发现 144s → 归来 36s |
