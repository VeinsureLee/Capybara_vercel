# M9: 手记模块 (`journal`)

> 每日旅行手记生成与展示，是旅行体验的核心产出和社交入口。

---

## 模块概览

| 属性 | 说明 |
|------|------|
| 职责 | 每日手记生成、匹配相遇段落、物品发现、手记展示 |
| 入口 | `/api/journal` (GET/POST) |
| 依赖 | AI Client, Supabase, Travel Module |
| 核心 | 旅行进行中 → 每日生成手记 → 有匹配时融入相遇叙事 |

---

## 手记内容结构

```
┌─────────────────────────────────┐
│  地点题头：今天在哪               │
├─────────────────────────────────┤
│  叙事主体：80-180字              │
│  卡皮视角，温暖治愈有画面感        │
├─────────────────────────────────┤
│  匹配相遇段落（如有）：60-120字   │
│  主题级披露，不露具体细节          │
├─────────────────────────────────┤
│  今日小物：一个探索物品            │
├─────────────────────────────────┤
│  交互区：                        │
│  · 进入对方河岸入口               │
│  · 分享按钮                      │
└─────────────────────────────────┘
```

---

## 叙事原则

- 第1天 = 新鲜好奇，中间 = 深入探索，最后一天 = 不舍离开
- 不流水账，有情绪弧线
- 卡皮语气：短句、画面感、偶尔跑题关注小东西
- 图内绝无文字（AI 生图约束），旁白独立排版

---

## 匹配相遇如何融入

| 条件 | 处理策略 |
|------|---------|
| 第1天 | 不触发相遇 |
| 相似度高 | 更深的情感共振、更具象的主题披露 |
| 相似度低 | 轻盈的同行温暖 |

手记末尾入口：
- "去Ta的河岸看看 →"
- "想多了解Ta一点吗？"

---

## 每晚推送

| 属性 | 说明 |
|------|------|
| 时间 | 20:00 ± 30分钟 |
| 截止 | 21:30 后不推送 |
| 频率 | 一天只一次 |

---

## 生成阻塞机制（V2.1 新增）

防止用户重复点击导致生成错误天数的手记。

**数据库约束：**
- journals 表加 `UNIQUE(travel_id, day_number)` 约束，数据库级防重复

**后端校验：**
- POST /api/journal 新增 `expected_day` 参数
- 后端计算 dayNumber 后与 expected_day 比对，不一致则返回 conflict 错误
- 已有的 409 检查保留（查询已有手记）

**前端防护：**
- 点击后按钮立即 disabled + 显示 loading 动画
- 请求完成前不可再次点击
- conflict 错误时自动刷新旅行状态、重算天数

## 数据结构

```typescript
interface Journal {
  id: string                          // UUID
  user_id: string                     // FK → auth.users
  capybara_id: string                 // FK → capybaras
  travel_id: string                   // FK → travels
  day_number: number                  // 旅行第几天
  location_name: string               // 地点名称
  narrative: string                   // 叙事主体（80-180字）
  encounter_narrative: string | null  // 匹配相遇段落（60-120字）
  matched_user_id: string | null      // 匹配到的用户
  item_found: {                       // 今日小物
    name: string
    description: string
    category: string
    rarity: string
  } | null
  visual_highlights: {                // V2.1 新增：图片标注数据
    keyword: string                   // 手记中的关键词
    description: string               // 简短描述（1行）
    suggested_position: string        // AI建议的大致位置区域
  }[] | null
  published_at: string                // 推送时间
  created_at: string
}
```

> **UNIQUE 约束：** `journals(travel_id, day_number)` — 同一次旅行的同一天只能有一篇手记

## 手记展示位置（V2.1 调整）

手记不再直接在探索主页面展示，而是放在单次旅行详情页（`/travel/[id]`）中。

- 探索主页只显示地图 + 探索记录摘要（几篇手记、几件物品）
- 点击探索记录进入详情页，查看完整手记列表
- 手记中的 `visual_highlights` 用于地点图片标注功能
