# M10: 经验成长模块 (`growth`)

> 定义卡皮巴拉经验获取来源、升级曲线、等级解锁能力，以及成长在产品体验中的呈现方式。核心原则：**成长是自然发生的陪伴副产品，不是需要追逐的数值目标。**

---

## 设计理念

| 原则 | 说明 |
|------|------|
| 陪伴优先 | 经验来自真实互动（聊天、旅行、记忆），不设签到/打卡等纯数值驱动行为 |
| 不量化关系 | 等级影响卡皮的能力解锁，但不影响匹配、不排名、不展示给他人 |
| 无焦虑感 | 不显示进度条/百分比，只在升级瞬间给出温暖提示；用户不需要知道"还差多少" |
| 不老不病不死 | 等级只增不减，没有衰退/惩罚机制，长期离线后回来一切如旧 |
| 避免氪金感 | 付费装扮不提供经验加成，等级解锁能力全部免费可达 |

---

## 经验获取来源

### 核心来源

| 行为 | 单次经验值 | 频率上限 | 说明 |
|------|-----------|---------|------|
| 对话（每轮） | +3 | 50轮/天 | 每条用户消息 + 卡皮回复算一轮 |
| 旅行完成 | +15 × 天数 | 无 | 1天旅行 = +15，5天旅行 = +75 |
| 记忆提取 | +10 | 跟随对话频率 | AI 从对话中成功提取出结构化记忆时触发 |
| 旅行手记生成 | +5 / 篇 | 跟随旅行天数 | 每日手记生成时获得 |

### 社交来源

| 行为 | 单次经验值 | 频率上限 | 说明 |
|------|-----------|---------|------|
| 旅途相遇（被匹配） | +20 | 1次/旅行 | 旅行中被记忆匹配系统配对时触发 |
| 串门完成（V1） | +10 | 无 | V1 串门系统完成一次拜访 |

### 探索来源（V1 兼容）

| 行为 | 单次经验值 | 频率上限 | 说明 |
|------|-----------|---------|------|
| V1 探索完成 | +10 / +20 / +30 | 无 | 短/中/长途探索 |
| 发现稀有物品 | +5 (uncommon) / +10 (rare) / +25 (legendary) | 无 | 按稀有度额外奖励 |

### 设计约束

- **不作为经验来源的行为**：登录/签到、分享、邀请好友、付费购买
- **每日经验上限**：300 EXP — 防止自动化刷经验，正常使用不会触达
- **离线期间**：不产生经验，不惩罚

## 升级曲线

采用**平方根减速曲线**：前期升级快、后期稳步变慢，但不会慢到令人绝望。

### 升级公式

```
升级所需经验 = FLOOR(BASE × level^EXPONENT)

BASE = 80
EXPONENT = 1.5
```

### 等级经验表

| 等级 | 累计经验 | 本级所需 | 约需天数（正常使用） |
|------|---------|---------|-------------------|
| 1 → 2 | 80 | 80 | ~1 天 |
| 2 → 3 | 306 | 226 | ~2 天 |
| 3 → 4 | 720 | 414 | ~3 天 |
| 4 → 5 | 1,358 | 638 | ~5 天 |
| 5 → 6 | 2,243 | 885 | ~7 天 |
| 6 → 7 | 3,397 | 1,154 | ~9 天 |
| 7 → 8 | 4,838 | 1,441 | ~11 天 |
| 8 → 9 | 6,580 | 1,742 | ~13 天 |
| 9 → 10 | 8,640 | 2,060 | ~16 天 |
| 10 → 11 | 11,030 | 2,390 | ~18 天 |

> **"约需天数"假设**：每天对话 20 轮(+60)、每 3 天完成一次 3 天旅行(+45 旅行 + 15 手记 + 10 记忆 = 均摊 ~23/天)、偶尔匹配(~3/天) → 日均约 86 EXP。

### 等级上限

暂定 **20 级**。达到 20 级后经验继续累积但不再升级，为未来版本保留扩展空间。

## 等级解锁能力

等级解锁的是卡皮的**行为丰富度**，不是数值优势。

| 等级 | 解锁内容 | 说明 |
|------|---------|------|
| 1 | 基础对话 + 短途旅行 | 初始状态 |
| 2 | 中途旅行（2-3天） | 卡皮愿意走得更远一点 |
| 3 | 记忆提取 | 卡皮开始记住你说的事情 |
| 4 | 长途旅行（4-5天） | 卡皮敢于远行 |
| 5 | 记忆匹配社交 | 可分享记忆参与全局匹配 |
| 7 | 生活层动作扩展 | 休息日更多闲暇动作（如发呆看星星） |
| 10 | 旅行偏好表达 | 卡皮会主动提出想去某类地方 |
| 15 | 手记风格个性化 | 手记叙事带有更多性格色彩 |
| 20 | 满级称号 | "老朋友"标记（仅自己可见） |

### 设计说明

- 等级 1-5 覆盖核心功能逐步开放，正常使用约 2-3 周可达，保证新手期有节奏感
- 等级 5 以后的解锁全部是体验增强，非必需功能
- **不解锁任何影响匹配/社交优势的能力** — 新用户和老用户在社交层面完全平等

## 升级体验呈现

### 升级时刻

升级时不打断当前操作，在下一次打开首页时展示：

```
"卡皮好像又长大了一点呢... 现在是等级 {N} 了"
```

配合卡皮动画（伸懒腰/甩尾巴），持续 2-3 秒后自然消失。

### 首页展示

- 仅显示当前等级数字，如 `Lv.5`
- **不显示**经验条、百分比、距下一级还差多少
- 点击等级数字可查看已解锁能力列表

### 不展示的内容

- 不展示经验值数字
- 不展示排行榜
- 不在社交/匹配场景中展示等级
- 不在手记/日志中提及等级

## 数据模型变更

现有字段已满足需求，无需新增表：

```sql
-- capybaras 表已有字段
experience INT DEFAULT 0,   -- 累计经验值
level      INT DEFAULT 1    -- 当前等级
```

### 需要新增的计算逻辑（实现时）

```typescript
// 升级阈值计算
function expForLevel(level: number): number {
  return Math.floor(80 * Math.pow(level, 1.5))
}

// 累计经验阈值
function totalExpForLevel(targetLevel: number): number {
  let total = 0
  for (let i = 1; i < targetLevel; i++) {
    total += expForLevel(i)
  }
  return total
}

// 检查是否应该升级
function checkLevelUp(currentLevel: number, totalExp: number): number {
  let level = currentLevel
  while (level < 20 && totalExp >= totalExpForLevel(level + 1)) {
    level++
  }
  return level
}
```

## 经验写入时机

经验增加应在以下 API 路由中实现：

| API 路由 | 触发点 | 经验值 |
|----------|--------|--------|
| `POST /api/chat` | AI 回复成功后 | +3 |
| `POST /api/chat` | 回复中包含 memory_extract 时 | +10（额外） |
| `GET /api/travel` | 惰性完成检测到旅行结束 | +15 × duration_days |
| `GET /api/travel` | 惰性完成时发现有 matched_user_id | +20（额外） |
| `POST /api/journal` | 手记生成成功 | +5 |
| `GET /api/explore` | V1 探索惰性完成 | +10/20/30 |
| `POST /api/visit` | V1 串门完成 | +10 |

每次经验写入后调用 `checkLevelUp()`，如等级变化则同时更新 `level` 字段。

## 与其他模块的关系

```
对话模块 ──(+3/轮, +10/记忆)──→ ┌──────────┐
旅行模块 ──(+15×天, +20/匹配)──→ │ 经验成长  │ ──→ 等级解锁能力
手记模块 ──(+5/篇)──────────────→ │  模块    │ ──→ 首页等级展示
探索模块 ──(+10~30)─────────────→ │          │ ──→ 旅行时长限制
串门模块 ──(+10)────────────────→ └──────────┘
```

- **对话模块**：最高频的经验来源，保证"只要聊天就在成长"
- **旅行模块**：受等级限制（Lv.2 解锁中途、Lv.4 解锁长途），也是重要经验来源
- **记忆模块**：Lv.3 解锁记忆提取，Lv.5 解锁记忆匹配社交
- **匹配模块**：不受等级影响匹配质量，但 Lv.5 以下不参与匹配

## 边界与风险

| 风险 | 应对 |
|------|------|
| 刷经验（自动化发消息） | 每日 300 EXP 上限 + 每日对话 50 轮上限 |
| 新用户被"等级门槛"劝退 | 核心功能 Lv.1 即可用，解锁节奏快（Lv.5 约 2-3 周） |
| 等级带来的"进度焦虑" | 不展示经验条/百分比，升级是惊喜而非目标 |
| 付费 vs 免费用户差距 | 付费仅影响装扮外观，经验获取完全平等 |
| 老用户 vs 新用户社交不平等 | 匹配算法不考虑等级，仅看记忆相似度 |
