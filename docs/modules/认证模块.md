# M1: 认证模块 (`auth`)

> 负责用户注册、登录、会话管理与路由保护，基于 Supabase Auth 和 `@supabase/ssr` 实现。

---

## 模块概览

| 属性 | 说明 |
|------|------|
| 职责 | 用户注册、登录、会话管理、路由保护 |
| 入口 | `/login`, `/register`, `middleware.ts` |
| 依赖 | Supabase Auth, `@supabase/ssr` |
| 数据流 | Browser → Supabase Auth → Session Cookie → Middleware 验证 |

---

## 关键流程

### 注册流程

```
用户输入 → supabase.auth.signUp(email, password)
  → profiles 表自动创建记录（数据库触发器）
  → 跳转 /chat
```

### 登录流程

```
用户输入 → supabase.auth.signInWithPassword(email, password)
  → Set-Cookie (sb-access-token, sb-refresh-token)
  → 跳转 /chat
```

### 中间件保护流程

```
Request → 读取 Cookie → supabase.auth.getUser()
  → 未登录且非公开路由 → redirect /login
  → 已登录且在登录页 → redirect /chat
```

---

## 配置项

| 配置 | 值 | 说明 |
|------|---|------|
| Email 确认 | 关闭 | MVP 阶段减少摩擦 |
| 会话有效期 | Supabase 默认 | 1 小时 access_token + 7 天 refresh_token |
| 密码策略 | 最少 6 位 | Supabase 默认 |

---

## 认证安全

| 措施 | 实现 |
|------|------|
| Session Cookie | httpOnly + Secure + SameSite=Lax |
| Token 刷新 | Middleware 每次请求自动刷新 |
| 密码存储 | Supabase Auth 管理（bcrypt） |
| API 鉴权 | 每个 Route 调用 `getUser()` 验证 |
