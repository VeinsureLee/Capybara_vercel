# M2: 卡皮巴拉模块 (`capybara`)

> 管理卡皮巴拉的创建、查询、状态流转和属性更新，是系统的核心数据实体。

---

## 模块概览

| 属性 | 说明 |
|------|------|
| 职责 | 创建、查询、状态管理、属性更新 |
| 入口 | `/api/capybara`, `/chat` 页面初始化 |
| 依赖 | Supabase Server Client |
| 约束 | 每用户唯一（UNIQUE owner_id） |

---

## 状态机

```
                ┌──────────┐
                │          │
     创建 ──→  │   home   │ ←── 探索完成
                │  (在家)   │ ←── 串门归来
                │          │
                └──┬───┬───┘
                   │   │
          探索出发  │   │  串门出发（未实现）
                   │   │
            ┌──────┘   └──────┐
            ▼                 ▼
     ┌──────────┐      ┌──────────┐
     │exploring │      │ visiting │
     │ (探索中)  │      │ (串门中)  │
     └──────────┘      └──────────┘
```

状态转换规则：

| 当前状态 | 事件 | 目标状态 |
|---------|------|---------|
| (无) | 用户创建 | home |
| home | 触发探索 | exploring |
| home | 触发串门（未实现） | visiting |
| exploring | 探索完成（惰性检查） | home |
| visiting | 串门归来（未实现） | home |

## 数据结构

```typescript
interface Capybara {
  id: string                                    // UUID
  owner_id: string                              // FK → auth.users
  name: string                                  // 用户命名
  personality_type: string                      // 'gentle'（默认）
  traits: string[]                              // ['治愈','淡定','好奇','友善']
  mood: string                                  // 当前心情
  experience: number                            // 经验值
  level: number                                 // 等级
  status: 'home' | 'exploring' | 'visiting'    // 状态
  memory: string[]                              // 长期记忆（关键词）
  created_at: string
}
```

## 心情系统（mood）

| 心情 | 触发条件 | 影响 |
|------|---------|------|
| happy | 正面对话、探索归来 | 对话更活泼，探索意愿高 |
| curious | 讨论新话题 | 容易触发探索 |
| sleepy | 长时间未互动 | 对话简短 |
| excited | 发现稀有物品 | 探索故事更丰富 |
| calm | 日常聊天 | 默认状态 |

心情由 AI 在每次对话中根据对话内容返回，写入 `capybaras.mood` 字段。

---

## 记忆系统（memory）

### 记忆层级

| 类型 | 存储位置 | 容量 |
|------|---------|------|
| 短期记忆 | conversations 表最近 20 条记录 | 20 条对话 |
| 长期记忆 | capybaras.memory JSONB 数组 | 最多 50 个关键词 |

### 记忆写入规则

```
每次对话后 AI 返回 keywords[] → 追加到 memory
memory 超过 50 时 → 移除最旧的（FIFO）
```

### 记忆读取规则

```
生成对话 prompt 时 → 注入最近 20 条对话 + memory 全部
生成探索 prompt 时 → 注入 keywords 加权聚合结果
```

## JSONB 字段结构

**`capybaras.traits`** — 性格标签数组：
```json
["治愈", "淡定", "好奇", "友善"]
```

**`capybaras.memory`** — 长期记忆关键词：
```json
["失眠", "星空", "小时候的事", "想远行", "最近压力大"]
```
