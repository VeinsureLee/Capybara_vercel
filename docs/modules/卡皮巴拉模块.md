# M2: 卡皮巴拉模块 (`capybara`)

> 管理卡皮巴拉的创建、查询、状态流转、属性更新、生活层动作和装扮系统，是系统的核心数据实体。

---

## 模块概览

| 属性 | 说明 |
|------|------|
| 职责 | 创建、查询、状态管理、属性更新、生活层调度、装扮管理 |
| 入口 | `/api/capybara`, `/chat` 页面初始化 |
| 依赖 | Supabase Server Client |
| 约束 | 每用户唯一（UNIQUE owner_id） |

## 状态机（V2）

```
                ┌──────────┐
                │          │
     创建 ──→  │   home   │ ←── 休息完成（1-2天后）
                │  (在家)   │
                │          │
                └────┬─────┘
                     │
            用户/卡皮触发旅行
                     │
                     ▼
              ┌──────────┐
              │traveling │     1-5 天
              │ (旅行中)  │     每天生成手记
              └────┬─────┘
                   │
          estimated_return 到达
          惰性完成检查
                   │
                   ▼
              ┌──────────┐
              │ resting  │     1-2 天
              │ (休息中)  │     生活层动作出现
              └────┬─────┘
                   │
            休息期满自动回到 home
                   │
                   └──→ home → 下一次旅行
```

状态转换规则：

| 当前状态 | 事件 | 目标状态 |
|---------|------|---------|
| (无) | 用户创建 | home |
| home | 用户/卡皮触发旅行 | traveling |
| traveling | 旅行完成（惰性检查，estimated_return <= now） | resting |
| resting | 休息期满（rest_end <= now） | home |

### V1 Legacy 状态（保留向后兼容）

V1 使用 `home ↔ exploring ↔ visiting` 三态模型。V2 中 `exploring` 和 `visiting` 状态仍被保留在代码中，供 `/api/explore` 向后兼容使用，但新流程统一走 `home → traveling → resting → home`。

| V1 状态 | V2 对应 | 说明 |
|---------|---------|------|
| exploring | traveling | V1 短时探索 → V2 多日旅行 |
| visiting | (废弃) | V1 串门机制，V2 由旅途相遇替代 |

## 性格系统（V2）

### 核心原则：AI 读历史，不做结构化性格数据

V2 不再使用 `personality_type` 或 `traits` 等结构化字段驱动卡皮性格。卡皮的"个性"完全由 AI 读取对话历史和互动记录自然生成。

```
对话历史 + 近期互动 + 相关记忆  →  AI 理解这只卡皮的性情  →  在所有输出中保持一致
```

设计理由：

- 当代大模型已足够强大，读 30 天对话即可准确把握卡皮的性情
- 结构化数值会让输出机械化，维度边界产生梯度跳跃
- 调优只需改 prompt，不用改代码
- 个性是"相处出来的"，不是"选出来的"——完全由用户互动历史塑造

> V1 Legacy: `personality_type` 和 `traits` 字段仍保留在数据库中，但不再作为 prompt 输入的主要性格来源。

## 意愿系统（V2）

### 行为词汇表

AI 需要理解和表达以下行为词汇（不是数据结构，由 AI 基于对话历史自然判断触发）：

| 行为 | 触发场景 | 大致频率 |
|------|---------|---------|
| 婉拒用户建议 | 用户让它去某处 / 做某事 | ~10% |
| 主动起话题 | 用户进对话界面停留不说话（仅 app 内） | 旅行日 ~30%、休息日 ~40% |
| 主动回忆过去 | 聊天中自然联想到用户说过的话 | 由 AI 掌握 |
| 表达记忆判断 | 遇到重要记忆时（"这个我想记着" / "这个就咱俩知道"） | 由 AI 掌握 |
| 装扮情绪反馈 | 用户穿上装扮 | 每次穿戴触发，三档 |
| 生活层动作选择 | 休息日展示 | 每 60-120 秒切换 |

### 统一原则

基于对话历史读出的卡皮性情、当前情境，AI 给出自然一致的表现。不需要每种行为单独写决策逻辑。

### 意愿 vs 隐私的双面性

| 后台（隐私系统） | 用户视角（卡皮的表达） |
|---------------|---------------------|
| AI 打标 shareable=true | 卡皮说"这件事挺有意思的，我想记着" |
| AI 打标 shareable=false（private） | 卡皮说"这个就你我知道" / "我不跟别人说" |
| 敏感话题，永不进入匹配池 | 卡皮表达关心，不追问、不转发 |

## 生活层（仅休息日出现）

卡皮在旅行间歇的休息日（每次旅行后 1-2 天）回家，河岸上展示 5 种日常动作循环：

| 动作 | 基础时长 | 默认时段偏好 |
|------|---------|------------|
| 睡 (sleep) | 90-180 秒 | 夜间、午后 |
| 游 (swim) | 60-120 秒 | 上午、傍晚 |
| 发呆 (idle) | 40-90 秒 | 全天 |
| 吃草 (eat) | 40-80 秒 | 上午、傍晚 |
| 看风景 (gaze) | 50-100 秒 | 清晨、黄昏 |

动作调度由 AI 基于时段、卡皮偏好（读对话历史）、用户是否在看综合判断。用户点击卡皮可进入对话。

## 装扮系统

### 槽位结构（4 槽位）

```
头部槽 (head)     — 帽子、发饰、面具
身体槽 (body)     — 围巾、披风、衣服
尾部槽 (tail)     — 尾饰、鞋袜
点缀槽 (accessory) — 项链、徽章、眼镜等小件
```

V1 内容极简（每槽 3-5 款），但结构定死，V2 扩内容时不改架构。

### 装扮情绪反馈

用户穿上装扮时，卡皮有情绪反馈（三档）：

| 情绪 | 表现 | 时长 |
|------|------|------|
| 开心 (happy) | 尾巴轻摇、嘴角上扬 | 2-3 秒 |
| 一般 (neutral) | 眨眼、微微点头 | 1-2 秒 |
| 不太舒服 (uncomfortable) | 耷拉、摇头、微蹩 | 2-3 秒 |

档位由 AI 基于装扮风格描述和卡皮对话历史（读出偏好）判断。V1 不做拒穿。

## 数据结构（V2）

```typescript
interface Capybara {
  id: string                                              // UUID
  owner_id: string                                        // FK → auth.users
  name: string                                            // 用户命名
  status: 'home' | 'traveling' | 'resting'                // V2 状态
       | 'exploring' | 'visiting'                         // V1 legacy 状态（向后兼容）
  mood: string                                            // 当前心情（AI 每次对话更新）
  experience: number                                      // 经验值
  level: number                                           // 等级

  // V2 新增字段
  equipped_costumes: {                                    // 当前穿戴装扮
    head: string | null                                   // 头部槽 → costume_id
    body: string | null                                   // 身体槽 → costume_id
    tail: string | null                                   // 尾部槽 → costume_id
    accessory: string | null                              // 点缀槽 → costume_id
  }
  current_life_action: string | null                      // 当前生活层动作（sleep/swim/idle/eat/gaze）
  last_travel_completed_at: string | null                 // 上次旅行完成时间（用于计算休息期）

  // V1 Legacy 字段（保留向后兼容，不再作为主要驱动）
  personality_type: string                                // V1: 'gentle' 等
  traits: string[]                                        // V1: ['治愈','淡定','好奇','友善']
  memory: string[]                                        // V1: 长期记忆关键词数组

  created_at: string
}
```

## 心情系统（mood）

| 心情 | 触发条件 | 影响 |
|------|---------|------|
| happy | 正面对话、旅行归来 | 对话更活泼，旅行意愿高 |
| curious | 讨论新话题 | 容易触发旅行 |
| sleepy | 长时间未互动 | 对话简短 |
| excited | 发现稀有物品 | 手记故事更丰富 |
| calm | 日常聊天 | 默认状态 |

心情由 AI 在每次对话中根据对话内容返回，写入 `capybaras.mood` 字段。

## 记忆系统（V2）

### V2 结构化记忆

V2 使用独立的 `memories` 表存储结构化记忆，替代 V1 的关键词数组。每条记忆由 AI 从对话中自动提取并分类。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| capybara_id | UUID | FK → capybaras |
| user_id | UUID | FK → auth.users |
| content | string | 记忆原文摘要 |
| topic | string | AI 归纳的主题标签（如"失去宠物"、"失眠"、"童年怀旧"） |
| shareable | boolean | 是否可分享（AI 自动分类，用户可手动修改） |
| sensitive | boolean | 是否为敏感话题（永不进入匹配池） |
| source_conversation_id | UUID | 来源对话 ID |
| created_at | timestamp | 创建时间 |

### 记忆分类规则

| 分类 | shareable | 说明 |
|------|-----------|------|
| 可分享 | true | 偏好、兴趣、情绪状态、一般性生活描述 |
| 私密 | false | 涉及具体人名/关系、健康、财务、法律、亲密关系细节 |
| 敏感 | false + sensitive=true | 自杀/自伤/严重心理危机等，永不进入匹配池 |

用户可在"卡皮记忆库"随时查看和切换每条记忆的可分享状态。

### 记忆读取规则

```
生成对话 prompt 时 → 注入最近 20 条对话 + 相关记忆
生成旅行 prompt 时 → 注入可分享记忆的主题标签用于地点匹配
匹配时 → 仅使用 shareable=true 且 sensitive=false 的记忆
```

### V1 Legacy 记忆系统

> V1 使用 `capybaras.memory` JSONB 数组存储关键词，容量上限 50 个，由每次对话 AI 返回的 `keywords[]` 追加写入。V2 中该字段保留但不再作为主要记忆来源。

## JSONB 字段结构

**`capybaras.equipped_costumes`** — 装扮槽位（V2 新增）：
```json
{
  "head": "costume_uuid_or_null",
  "body": "costume_uuid_or_null",
  "tail": null,
  "accessory": "costume_uuid_or_null"
}
```

**`capybaras.traits`** — 性格标签数组（V1 Legacy）：
```json
["治愈", "淡定", "好奇", "友善"]
```

**`capybaras.memory`** — 长期记忆关键词（V1 Legacy）：
```json
["失眠", "星空", "小时候的事", "想远行", "最近压力大"]
```
