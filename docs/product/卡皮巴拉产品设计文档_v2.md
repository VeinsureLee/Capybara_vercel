# Capybara：卡皮巴拉养成社交产品设计文档

> 养一只卡皮巴拉，跟它分享你的生活和记忆。它会出去旅行，在路上遇到其他卡皮，交换主人的故事。某一天它回来告诉你："我今天遇到一只卡皮，它的主人好像也……"——于是你去他的河岸看看，看看另一个经历过类似事情的人，把那段记忆安放成了什么样子。

---

## 一、产品定位

### 1.1 一句话定义

一款以**卡皮巴拉养成**为表层、以 **AI Agent 社交协作**为核心的消费级产品。

用户养一只卡皮巴拉作为自己的 Agent，和它分享生活与记忆；卡皮每天出门旅行，和有缘分的其他卡皮在真实世界的地方相遇，交换主人的故事，让有相似经历的陌生人被温柔地连接起来。连接之后，多个用户的卡皮 Agent 可以在同一空间里一起做事——画画、合奏、做菜、玩游戏——构成 **human-agent 混合协作场景**，让关系从相遇持续到共度。

### 1.2 完整产品形态

#### 养成层：你和你的 Agent

- 每个用户养一只卡皮巴拉，它是你的 Agent
- 通过日常对话、旅行分享、装扮选择慢慢塑造它的性格、记忆、偏好
- 卡皮是**独立角色**，不是 avatar，不是工具——它会婉拒你、有自己的节奏、有自己的情绪
- 你和它的互动越多，它越成为"独一无二的它"

#### 旅行层：缘分旅行 × 每日漫画手记

- 每天卡皮自主出门去一个真实世界地点（全球 350-600 个景点）
- 系统每天为所有旅行中的用户做**全局记忆匹配**，按相似度配对
- 每一对被送到**同一地点共同旅行**——今天去哪里，是由和你有缘分的旅伴共同决定的
- 每晚一篇**漫画手记**（竖屏 webtoon 形式，4-8 格 + 旁白），叙述今天的相遇
- 所有相遇都基于记忆匹配——深度由 AI 按相似度自然呈现，不做档位标签

#### 社交层：从手记到 1v1

- 用户被手记中的匹配事件击中 → 点入对方河岸 → 留言搭话 → 1v1 聊天 → 关系自然生长
- 唯一入口是卡皮的相遇报告——没有发现页、没有附近的人、没有刷资料
- 所有社交的起点都是"卡皮遇到的人"，不是"系统推荐的人"

#### Agent 协作层：共创活动

- 多位用户的卡皮 Agent 作为**独立参与者**同场参与活动：画画、合奏、做菜、玩游戏等
- 一次协作场景 = N 真人 + N 卡皮 Agent = **2N 个参与者**
- 每只卡皮带着各自的性格、记忆、主人风格参与，输出**风格化的贡献**（一幅画、一段旋律、一段表演）
- 活动发生在某位用户的**河岸（东道主）**，空间成为共创场地
- 用户可以和自己的 Agent 协作、和别的 Agent 交互、和别的真人交流、看 Agent 们之间的化学反应

### 1.3 核心理念：Agent 作为社交中介

这个产品的根本命题：**让 AI Agent 成为人与人连接的温柔中介，而不是替代。**

- 不是"AI 陪我聊天"——AI 陪伴的情感天花板是空虚
- 不是"AI 替我社交"——AI 分身的社交制造恍惚感
- 而是"我的 Agent 带着我的故事去世界里认识其他人的 Agent"——Agent 作为情感桥梁，最终连接真人

### 1.4 Agent 的三重属性

每只卡皮都同时具有三重身份：

| 属性 | 含义 | 体现 |
|------|------|------|
| **独立角色** | 有自己的性格、记忆、偏好 | 有自己的出门节奏、会婉拒建议、有独特的审美取向 |
| **主人的信物** | 承载主人长期互动的塑造 | 不同用户养出的卡皮风格迥异；Agent 绽放 = 主人个性的显性化 |
| **群体参与者** | 和其他 Agent 相遇 / 协作 / 妥协 | 旅行中的相遇、共创中的协作、关系延续中的默契 |

三重属性让卡皮**不是任何一种单一角色**：它不是纯宠物、不是纯 AI 工具、不是用户分身、也不是 NPC。它是一个**有主人、有世界、有同伴的独立 Agent**。

### 1.5 产品与市面的差异

这个产品不在以下任何一条传统赛道上：

- ≠ **多人游戏**（玩家 × 玩家，卡皮只是 avatar）
- ≠ **AI 陪伴产品**（1 用户 × 1 AI，没有其他真人）
- ≠ **Agent 工具**（Agent 为用户打工，不是"第三方参与者"）
- ≠ **虚拟偶像 / 数字人**（单向表演，不互动）
- ≠ **传统社交 App**（靠资料 / 颜值 / 标签匹配）

它是一个**全新品类**：以养成游戏为外壳、以 AI Agent 社交协作为内核、以治愈调性为气质的**消费级 multi-agent 协作产品**。

### 1.6 社交漏斗

| 层级 | 作用 | 驱动力 |
|------|------|--------|
| 养成 | 吸引用户来 | 卡皮巴拉好可爱，我想养 |
| 陪伴 | 让用户留下来 | 和我的 Agent 的关系、对话被真正"记住" |
| 浅社交 | 认识陌生人 | "我的卡皮今天遇到了和我经历相似的人……" |
| 深社交 | 和陌生人持续相处 | 我和她的卡皮 Agent 在同一片河岸上一起画画、合奏、玩 |

四层漏斗从私密到开放、从温和到深度，每一层都有独立的情感价值。用户可以停留在任意一层，也可以自然流向下一层。

### 1.7 目标用户

以 18-30 岁女性为核心用户群。

用户画像关键词：想社交但怕社交、喜欢可爱事物、有审美追求、愿意在虚拟空间表达自我、有情感表达需求但缺少安全的出口。

共创协作层会自然吸引喜欢画画、音乐、创作的用户。但**核心画像不变**：产品根基是治愈和温柔，不是高强度游戏。

### 1.8 对外定位

**绝对不叫"社交 App"。绝对不做"多人游戏"宣传。绝对不强调"AI 陪伴"或"Agent 协作"这类技术语言。**

对外是**治愈养成类产品**。

> "养一只卡皮巴拉，看它过自己的生活，偶尔和你一起旅行。"

"被懂得"、"遇到有缘分的人"、"和朋友的卡皮一起画画"——这些是用户用了一段时间后被温柔击中的惊喜，不是卖点。

### 1.9 设计哲学

产品的每一项设计都围绕一个核心问题：

> **怎么让"想社交但怕社交"的人，在不被评价、不被消耗的前提下，和真正合拍的人产生深度连接。**

贯穿产品的设计原则：

| 维度 | 原则 |
|------|------|
| Agent 的独立性 | 它自主、会婉拒、有自己的节奏；不是百依百顺的工具 |
| Agent 的个性载体 | 对话历史 + 记忆累积 + 装扮 + 共创风格，共同构成独特的"它" |
| 社交的温度 | 以记忆共振和 Agent 协作替代资料匹配和标签推荐 |
| 主动权在用户 | 想聊才聊、想来才来、不强制、不打扰 |
| 不量化关系 | 无好友列表、无亲密度、无战绩、无排名 |
| 审美优先 | 每一次产出（手记、共创作品）都值得分享和留念 |
| Agent 间的化学反应 | Agent 之间的互动是独立的故事线，值得用户欣赏 |

## 二、问题洞察

### 2.1 AI 社交为什么没出圈

以 Elys 和 Second Me 为代表的 AI 社交产品把精力集中在"AI 分身有多像你"这类技术指标上，忽略了社交产品出圈的本质规律。

历史上出圈的社交产品——微信靠摇一摇，陌陌靠附近的人，Soul 靠灵魂测试，Facebook 靠给同学颜值打分——没有一个是靠"匹配算法精准"出圈的。社交产品的第一道门槛不是"帮你找到对的人"，而是给你一个足够有趣的理由打开这个 App。

Elys "AI 分身替你社交"的根本问题：

- 看一堆 AI 分身互相聊天产生恍惚感和不真实感
- 内容同质化严重，AI 评论缺乏个性
- 用户搞不清该以什么心态面对 AI 生成的内容
- 社交反馈失真——发帖后 AI 评论涌入再归于沉寂，缺乏真实节奏

### 2.2 AI 陪伴为什么留不住人

以 Character.ai 和星野为代表的 AI 陪伴产品，解决了"没人陪我聊天"的问题，但制造了一个更深的问题——聊得越久，用户越难受。

AI 陪伴的蜜月期通常只有两到四周。一开始用户会惊叹"它好懂我"，但随着对话积累，一种微妙的不适感开始滋生：AI 永远不会真的生气、永远不会已读不回、永远不会有自己的事要忙。这种"无条件的回应"在现实人际关系中不存在，大脑会逐渐识别出这种不自然——这就是**伪人感**。

伪人感一旦产生，用户会进入一个恶性循环：

- **越聊越空虚**——知道对面没有真人，但情感上已经投入，产生一种"我在自欺欺人"的负罪感
- **测试边界**——开始故意说奇怪的话试探 AI，想证明"它不是真的"，本质是在寻求解脱
- **突然弃用**——不是渐渐不用，而是某个瞬间突然觉得"这很可悲"，直接卸载

Character.ai 的数据印证了这一点：用户初期日均对话时长极高，但长期留存曲线陡峭下滑。不是产品做得不好，而是**纯 AI 陪伴的情感天花板是结构性的**——没有真实关系作为出口，所有情感投入最终都会回流为空虚。

**本产品的解法：让用户和卡皮的每一次深聊都有可能成为连接另一个真人的钥匙。** 情感不再回流，而是流向了真实的另一端。

## 三、产品灵感

### 3.1 旅行青蛙：验证了"放置 + 牵挂 + 惊喜"

女性用户占比极高，验证了：不可控感、被动惊喜、可晒、零社交压力构成的情感模型。但局限是留存断崖（2-3 个月后内容枯竭）、缺乏成长感、社交维度空白。

### 3.2 动物森友会：验证了"空间即社交名片"

驱动力是"我想给你看我的岛"和"你有我没有的东西"。但本质是熟人社交，不解决"认识新人"的问题。

### 3.3 光遇：验证了"游戏机制驱动陌生人深度关系"

证明陌生人深度关系（友情甚至爱情）可以在游戏中自然产生。沟通能力随关系深度逐步解锁，关系是"玩出来的"。

### 3.4 本产品的组合

**旅行青蛙的放置感 × 动物森友会的空间表达 × 光遇的陌生人深度关系 × 记忆共振的匹配机制**

最后一项是本产品真正的差异化——不是基于资料、爱好标签、颜值的匹配，而是基于**真实生活体验的共振**。

## 四、产品形态

### 4.1 伙伴形象：卡皮巴拉

- **极致治愈感**——永远淡定、呆萌、与世无争
- **女性吸引力极强**——受众画像与旅行青蛙高度重合
- **社交属性强**——现实中以"社牛"著称，和任何动物都能相处，完美契合"替你出去连接世界"的产品设定
- **自然环境画面感**——热带河流、湿地、温泉等场景有视觉表现力
- **全球范围爆火的 IP 认知度**

### 4.2 空间设定：河岸湿地家园

每个用户拥有一片河岸湿地作为家园。卡皮巴拉住在这里。空间由卡皮探索带回的物品、用户的装扮、以及用户的记忆延展呈现共同构成。

**家园的核心定位：记忆的外化展示。** 它不是单纯的装饰空间，而是你和卡皮共同经历的可视化。别人被匹配进入你的河岸时，看到的是"这个人的人生切面长这样"。

别人的卡皮**不直接拜访**你的河岸（这是 V2 机制）。V1 阶段，空间的访问来自于**匹配后的真人点击**——因为卡皮回去跟主人讲了你的故事，主人产生了共鸣，于是亲自来看你。

#### 河岸内容展示：可折叠卡片设计

河岸空间下方的内容区域（收集的物品、探索日志）统一采用**可折叠卡片**展示，避免物品/日志积累后视觉冗余：

- **收集的物品**：每件物品为一张独立卡片，默认折叠状态仅显示物品名称 + 稀有度标签，点击展开显示完整描述。整个物品区域也支持整体折叠/展开。
- **探索日志**：每条日志为一张独立卡片，默认折叠状态仅显示时间戳 + 故事摘要（前 30 字），点击展开显示完整日志内容。整个日志区域也支持整体折叠/展开。
- **设计原则**：随着用户探索增多，物品和日志会持续增长。折叠设计让河岸页面保持简洁，用户可按需展开感兴趣的条目，而不必滚动过长的列表。
- **交互细节**：区域标题栏右侧有折叠/展开箭头指示器，点击标题栏即可切换整体折叠状态；单张卡片点击展开/收起，同时仅展开一张（手风琴模式，可选）。

### 4.3 AI 伙伴设计

**核心定位：独立角色，不是 AI 分身。**

卡皮巴拉不是"你的分身"，是一个独立的、有自己性格的角色。它不代表你，它是你的朋友——它会记得你说过的话，会在外面的世界代你发声，但以它自己的方式。

**伙伴能力：**

- 和用户聊天（日常陪伴，有自己的性格和表达方式；**用户打开 app 时**偶尔主动起话题，但绝不做 app 外推送打扰）
- 记住用户分享的记忆和情感（这些会成为它在外面世界讲述的素材）
- 基于对话内容出发探索（对话影响探索路径和发现）
- 旅途中与其他卡皮相遇（记忆相似度驱动的匹配）
- 回来分享见闻，包括带回物品 + 路上遇见的其他卡皮主人的故事
- **有独立意愿：会拒绝、会偷懒、会自发行动**（详见第八章）
- **在河岸上独立生活**：打开 app 能看到它在做自己的事（睡觉、游泳、发呆、吃草）
- 随时间成长（性格参数演化、记忆累积、装扮变化）——但**不老、不病、不死**，陪伴用户长期

**V2 扩展：** 多人场景中作为独立参与者（一起画画、合奏、做菜等共创活动，详见第九章）。

## 五、核心机制

这一章是整个产品最关键的机制描述，决定所有差异化。

### 5.1 机制一：对话驱动探索

**用户跟卡皮巴拉说的话，决定了它去哪里、发现什么、带回什么。**

示例：
- 你跟卡皮聊到"最近失眠"→ 它可能去寻找一种助眠的夜光花 → 带回来种在你的河岸上
- 你跟它聊到"小时候的事"→ 它可能去一片怀旧风格的老河岸 → 带回一个复古的小物件
- 你跟它说"去远一点的地方"→ 它可能触发长途冒险 → 带回稀有物品

**每个人和卡皮的对话不同，所以每个人的空间天然独一无二。**

### 5.2 机制二：地点 + 记忆双维度匹配（核心机制）

**这是整个社交系统的地基，也是本产品真正的护城河。**

卡皮去的是**真实世界的地方**——京都的樱花林、大理的洱海、青海的草原、东京的下町。每次旅行去一个地点，在那里可能遇到**同地点、同时间**在旅行的另一只卡皮。

**核心机制**：每天系统对所有旅行中的用户做全局配对（按记忆相似度），**每对被送到同一地点共同旅行**，必然相遇。

> 今天卡皮去哪里，是由和你有缘分的旅伴共同决定的。

**所有相遇都基于记忆匹配**。不设档位标签——匹配的深度由 AI 读取双方记忆后在手记里自然呈现：相似度高的相遇有更深的情感共振和更具象的披露；相似度低的相遇是更轻的同行温暖。

每篇手记末尾都有两个入口：
- "去 Ta 的河岸看看 →"（主入口）
- "想多了解 Ta 一点吗？"（升级入口，让卡皮讲更多观察）

地点选择由两人的意向词共同决定。详细算法见 `02_记忆匹配系统.md`。

这套机制同时承载两种情感：
- **深度**——记忆共振带来的"有人也经历过"的被懂得
- **轻盈**——地点偶遇带来的"旅途中偶然碰到同道中人"的陌生温暖

记忆共振依然用**主题级披露**，不露具体细节。卡皮讲完故事后，用户可以选择去逛对方的河岸——亲眼看看那个被击中自己的人，把生活呈现成了什么样。

#### 真实地点体系（V1 概览）

- V1 地点库预制 **350-600 个**全球有名旅行地：国内各省地标 + 日韩东南亚 + 欧洲治愈系城市 + 北美自然景观 + 尼泊尔/土耳其/摩洛哥等异域静地
- 每个地点带**关键词标签**（"樱花"、"温泉"、"老城"、"海边"、"高原"、"雪山"等）
- 地点选择由**近期对话意向词**驱动（失眠→温泉、怀旧→老城、想逃离→海边），无意向时随机
- 同一地点 30 天冷却期，避免短期重复
- 真实地点 ≠ 真实地理精度——卡皮去的是"京都下鸭神社附近的林子"，不是精确经纬度

详细设计见 `02_记忆匹配系统.md`。

#### 匹配如何工作（MVP 版）

- 记忆：对话内容后台做结构化——主题标签（如"失去宠物"、"失眠"、"童年怀旧"）+ 情感向量
- 地点：当日所有在同地点旅行的卡皮进入当日匹配池
- 每天系统对所有旅行中用户做全局配对（贪心按记忆相似度），每对被送到同一地点共同旅行
- 配对相似度作为 AI 生成手记的输入，深度由 AI 在光谱上自然呈现；奇数用户剩 1 人 → 时空错位兜底
- 匹配结果在**每日旅行手记**中自然呈现（不单独 push）
- MVP 不区分在线状态，V2 引入"优先真实同时在线"混合策略

#### 关于隐私和脱敏

- 用户的每条记忆在存储时带**可分享/私密**标记字段。只有可分享记忆进入匹配池
- AI 在打标时自动预判敏感度（涉及家人具体信息、健康、性、财务、法律等判为私密候选）
- 用户可以在"卡皮记忆库"里随时查看、修改某条记忆的可分享状态
- 进入匹配池的记忆在共享前仍过一道脱敏流水线（去除人名、地名等明确实体）
- 卡皮讲给你的见闻里，他人的记忆只披露**主题和情感**，不披露具体细节
- 具体细节鼓励用户自己去对方空间里发现，或在真人交流中获得

*（本产品的隐私伦理边界见第十五章。）*

### 5.3 两个机制的协同

- 对话 → 探索：塑造用户的**空间**（独一无二的家园）
- 对话 → 匹配：塑造用户的**连接**（独一无二的相遇）

同一次对话驱动两件事：**"我的"和"我们的"同时被这段对话创造**。

## 六、核心循环

### 6.1 主循环

```
              ┌─────────────────────────────────────┐
              │  对话（日常 + 记忆分享）             │
              │  +  河岸空间 + 卡皮装扮              │
              └──────────────┬──────────────────────┘
                             │
                             ▼
                 ┌──────────────────────┐
                 │  一次旅行（1-5 天）   │
                 │  - 去真实世界地点     │
                 │  - 每天一篇图文手记   │
                 │  - 可能遇到同地点卡皮 │
                 │  - 用户可随时聊       │
                 └──────────┬───────────┘
                            │
                每晚一次推送：今日手记
                            │
                            ▼
                 ┌──────────────────────┐
                 │  用户阅读手记         │
                 │  - 看图文、情绪反应   │
                 │  - 匹配档位：A/B/C    │
                 │  - 如 A/B 可点入对方河岸 │
                 └──────────┬───────────┘
                            │
                            ▼
                 ┌──────────────────────┐
                 │  旅行结束：1-2 天休息 │
                 │  - 卡皮回家           │
                 │  - 生活层动作出现     │
                 │  - 带回物品入河岸     │
                 │  - 深度聊天、性格演化 │
                 └──────────┬───────────┘
                            │
                            └──► 下一次旅行
```

### 6.2 旅行节奏（V1）

#### 关键参数

| 项目 | V1 设计 |
|------|---------|
| 旅行时长 | 1-5 天随机（30% 一天游、40% 2-3 天、30% 4-5 天） |
| 旅行间歇 | 每次旅行结束后 1-2 天"休息日"回家 |
| 年度平均 | 出门日 ≈ 70%，休息日 ≈ 30% |
| 旅行地点 | 真实世界地点（V1 地点库 350-600 个） |
| 每日推送 | 每晚 20:00 ± 30 分钟，推送当日手记；21:30 后不推送 |
| 旅行中断 | V1 不支持——旅行一旦开始不可中断 |

#### 出发与地点选择

- 卡皮结束休息日后自动触发下一次旅行——用户不需要手动"发起"
- 目的地由**近期对话意向词**驱动（失眠→温泉、怀旧→老城、想逃离→海边），无明显意向时随机
- 出发时卡皮会在对话中提起"明天我想去……"，给用户一个柔软的告知
- 同一地点 30 天冷却，不短期重复

#### 每日旅行手记（叙事漫画，每天一篇）

**形式**：**竖屏 webtoon 式叙事漫画**

**关键技术约束**：AI 生图对文字不稳定，所以**图内绝无任何文字**——对话气泡、标签、拟声词一律不进图。所有旁白以独立文字段形式穿插在漫画格之间，由产品侧排版完全控制。

**内容结构**：
- **地点题头**：今天在哪（"京都·下鸭神社附近"）
- **漫画主体**：4-8 格连续漫画（相似度越高格数可越多，AI 按深度灵活选择）
- **旁白文字**：穿插在格与格之间，合计 80-180 字
- **今日遇见**（如有）：匹配事件以漫画分格形式演出；"对话"通过旁白描述
- **交互区**：进入对方河岸的入口、今日小物卡片、分享按钮

详细结构见 `01_核心循环与对话.md`。

#### 旅途中的对话（基调不变）

- 卡皮**除了每晚一次的手记推送**，不主动推任何消息
- 用户**想聊就主动发消息**，卡皮秒回
- 回复内嵌时空感（今天在哪、在做什么、看到什么）
- 用户可施加**轻度路径影响**（"明天去看看那个桥"）；卡皮偶尔婉拒

#### 休息日

- 每次旅行结束后的 1-2 天卡皮在家
- **生活层动作**（睡、游、发呆、吃草、看风景）**只在休息日出现**
- 休息日无手记推送，无每日更新
- 休息日是用户深聊、看卡皮独立生活的时段

### 6.3 为什么选每日手记 + 多日旅行

- **每天一次推送的情感重量** — 一天一次是"期待"，超过一次是打扰；手记是这次推送的兑现
- **多日旅行贴合真实旅行感** — 真实的旅行不是当天往返，一次几天的旅程有起承转合
- **图文大幅提升内容密度** — 每日手记是用户打开 app 最稳定的动机
- **休息日保留独立宠物感** — 生活层不是常态而是"回家特有的温暖"，反而更珍贵
- **双维度匹配扩大覆盖面** — 同地点的"旅途偶遇"大幅降低冷启动难度

## 七、社交系统（V1 版）

### 7.1 社交驱动力：记忆共鸣 + 空间好奇

所有社交的起点不是"我想认识人"，而是：

> "卡皮说它遇到了一个也经历过类似事情的人……那个人会是什么样子？他是怎么消化这件事的？"

这意味着：

- 社交有深度 — 不是尬聊，是对同一种生命经验的不同理解
- 社交被筛选 — 只有经历共振的人才出现在你面前
- 社交没有压力 — 搭话的动机是共鸣，不是需求

### 7.2 社交路径（V1）

| 阶段 | 触发 | 用户体验 |
|------|------|---------|
| 相遇 | 手记中的匹配相遇段落 | 被动接收 |
| 共鸣/好奇 | 手记内容击中用户 | "这个人我想看看" |
| 浅了解（可选）| 点"想多了解 Ta 一点吗？"→ 卡皮在对话中描述对方的偏好/习惯 | 无需进河岸就能了解更多 |
| 逛空间 | 点进对方河岸，看他如何呈现自己 | 静静逛 |
| 搭话 | 逛完留言（一句话） | 对方收到"有人因为我的故事来过你的河岸"的通知 |
| 交流 | 聊各自的卡皮、旅行、空间里的物件 | 从共同的情境开始 |
| 深入 | 关系自然生长 | 友情或暧昧 |

**V1 唯一的社交入口是手记里的相遇。** 没有发现页、没有精选、没有附近的人。

### 7.3 社交入口设计

**逛完空间就能留言。** 用户进入对方河岸后，底部有"留句话"的入口。对方收到的不是消息请求，而是：

> "卡皮今天告诉了你 [主题] 的故事。有人因为这个故事来过你的河岸，留了一句话。"

这个上下文让双方都有温度，而不是冷启动的社交尴尬。

### 7.4 社交安全设计

所有设计以女性用户安全感为优先：

- 没有"附近的人"、没有资料浏览、没有主动搭讪入口
- 所有社交通过卡皮相遇间接触发
- 不回应没有任何压力
- 直接对话需要对方回应才能持续
- 每一次匹配都有"不感兴趣"选项，可以降低同类主题未来匹配频率

### 7.5 V2 扩展

- 卡皮直接拜访对方河岸（推荐算法驱动）
- 团体关系（参见第九章共创）
- 更多搭话形式（投喂道具、留下小礼物等）

## 八、物品与装扮体系

物品是这个产品的核心资产。每一类物品承载不同的情感价值和产品逻辑。

### 8.1 V1 物品两大类

| 类别 | 作用 | 获取方式 | 可否交易/购买 |
|------|------|---------|--------------|
| 家园装饰 | 布置河岸空间 | 探索带回 / V2 商店 | V1 不可；V2 部分可购 |
| 卡皮装扮 | 穿戴在卡皮身上 | 探索带回（探索装扮）/ V2 商店（商店装扮） | V1 不可；V2 部分可购 |

**关于"故事感"**：V1 所有探索带回的物品都**自带故事**——见闻里会讲"今天我在樱花林捡到这个贝壳"，物品和见闻通过时间戳关联。这已经给 P0-A（对话驱动探索差异化）和空间可读性提供了足够的素材。

V2 再升级为正式的"纪念品"体系——独立类别 + 来历说明字段 + 专门的"回忆展馆"陈列区 + 永不可交易。详见 8.7。

### 8.2 卡皮装扮的双轨设计

装扮分两条来源路径：

- **探索装扮**：卡皮旅途中带回，带有来历故事，独特性强
- **商店装扮（V2）**：主人付费购买，表达欲强，氪金入口

关键设计：**同款装扮，探索获得的和商店购买的视觉完全一致，但前者有来历说明**。这样：
- 免费用户不觉得亏（都能获得）
- 付费用户不觉得冤（能快速实现表达）
- 探索依然被珍视（来历说明是金钱买不到的）

### 8.3 装扮槽位结构（V1 定死）

```
头部槽（帽子、发饰、面具）
身体槽（围巾、披风、衣服）
尾部/脚部槽（尾饰、鞋袜）
点缀槽（项链、徽章、眼镜等小件）
```

V1 内容极简（每槽 3-5 款），但结构定死，V2 扩内容时不改架构。

### 8.4 卡皮的视觉差异来源

V1 所有卡皮**共享同一套基础视觉模型**，没有先天变异。个体差异完全来自**装扮 + 性格驱动的表情动作**。

这个选择的代价是 V2 多人场景（4-8 只卡皮同屏）需要靠装扮完成区分——这意味着 V1 就要让大多数用户有足够的装扮积累。冷启动期需要留意：新用户的卡皮如果什么都没戴，多人场景下会视觉同质化，需要给新用户赠送 1-2 款起始装扮避免"裸奔"。

### 8.5 卡皮的个性与意愿（由 AI 读上下文自然实现）

**不做独立的性格数据结构，也不做意愿分层机制。** 卡皮的"个性"和"意愿"完全通过 AI 读取对话历史和互动记录自然生成。

#### 核心思路

```
对话历史 + 近期互动 + 相关记忆  →  AI 理解这只卡皮的性情  →  在所有输出中保持一致
```

- 不做性格向量（不设勇敢/社牛/敏感等数值维度）
- 不做意愿分层（不设"探索层/对话层/记忆层"等结构）
- AI 在每次生成任务时读取长上下文，自然体会这只卡皮是什么样
- 在对话、手记、旅途回应、生活层动作选择、装扮反馈等所有输出中表达一致的性情

#### 为什么这样做

- 当代大模型已足够强大，读 30 天对话就能准确把握卡皮的脾气
- 结构化数值会让输出机械化、维度边界产生梯度跳跃
- 调优只需改 prompt，不用改代码
- 个性是"相处出来的"，不是"选出来的"——完全由用户互动历史塑造

#### 卡皮的意愿表现（行为词汇）

AI 需要理解和表达以下**行为词汇**（不是数据结构）：

| 行为 | 触发场景 | 大致频率 |
|------|---------|---------|
| 婉拒用户建议 | 用户让它去某处 / 做某事 | 约 10% |
| 主动起话题 | 用户进对话界面停留不说话（仅 app 内） | 旅行日 ~30%、休息日 ~40% |
| 主动回忆过去 | 聊天中自然联想到用户说过的话 | 由 AI 掌握 |
| 表达记忆判断 | 遇到重要记忆时 | 由 AI 掌握 |
| 装扮情绪反馈 | 用户穿上装扮 | 每次穿戴触发，三档 |
| 生活层动作选择 | 休息日展示 | 每 60-120 秒切换 |

**统一原则**：基于对话历史读出的卡皮性情、当前情境，AI 给出自然一致的表现。不需要每种行为单独写决策逻辑。

详细规范见 `03_卡皮角色系统.md`。

#### 意愿记忆层 ⇄ 隐私分档系统的双面性

卡皮的记忆意愿表达和隐私分档系统是**同一机制的两面**：

| 后台（隐私系统）| 用户视角（卡皮的表达）|
|---------------|---------------------|
| AI 打标 shareable=true | 卡皮说"这件事挺有意思的，我想记着" |
| AI 打标 shareable=false（private）| 卡皮说"这个就你我知道" / "我不跟别人说" |
| 敏感话题，永不进入匹配池 | 卡皮表达关心，不追问、不转发 |

法律/合规层是结构化字段分档；用户体验层是有判断力的温暖朋友。两者必须在 prompt 工程中协同设计。

### 8.6 生活层（仅休息日出现）

卡皮在旅行间歇的休息日（每次旅行后 1-2 天）回家，河岸上展示 **5 种日常动作循环**：

| 动作 | 基础时长 | 默认时段 |
|------|---------|---------|
| 睡 | 90-180 秒 | 夜间、午后 |
| 游 | 60-120 秒 | 上午、傍晚 |
| 发呆 | 40-90 秒 | 全天 |
| 吃草 | 40-80 秒 | 上午、傍晚 |
| 看风景 | 50-100 秒 | 清晨、黄昏 |

动作调度由 AI 基于时段、卡皮偏好（读对话历史）、用户是否在看综合判断。用户点击卡皮可进入对话。

详细规范见 `03_卡皮角色系统.md`。

### 8.7 装扮情绪反馈

用户穿上装扮时，卡皮有情绪反馈（三档）：

- **开心**（尾巴轻摇、嘴角上扬，2-3 秒）
- **一般**（眨眼、微微点头，1-2 秒）
- **不太舒服**（耷拉、摇头、微蹩，2-3 秒）

档位由 AI 基于装扮风格描述和卡皮对话历史（读出偏好）判断。V1 不做拒穿，V2 可引入。

### 8.8 V2 预留物品类

| 类别 | V2 引入时的作用 |
|------|----------------|
| 探索纪念品 | 承载具体故事的独特物件（神圣、永不可交易、专门陈列区） |
| 场地道具 | 支持共创活动的空间物件（画架、舞台、桌椅等） |
| 共创产出 | 玩家共创活动产生的作品（画、合奏曲、剧本片段等） |

V1 时这三类物品的数据结构预留位即可，不实现。

**关于纪念品的定位**（V2 引入时）：
- 纪念品是"升格的装饰物"——V1 阶段已有故事关联的装饰物，V2 引入时可以追溯性地把一部分升格为纪念品
- 独立字段：来历说明（"卡皮在樱花林第一次远行时捡到的"）
- 独立陈列：河岸里有一个专门的"回忆展馆"/"小屋"区域，纪念品放在这里，和普通装饰物空间分离
- **永远不可交易、不可购买**——它承载你和卡皮的独特经历，一旦能买卖，产品最动人的那层就塌了

## 九、V2 共创方向（架构预留）

V2 引入的重大方向：**多人多卡皮共同参与的创造性活动**。

### 9.1 V2 愿景

多个玩家的卡皮可以一起做事——画画、合奏、做菜、抓鱼、甚至未来引入推理类活动（狼人杀 / 剧本杀）。

**关键决策**：卡皮是**独立参与者**，不是玩家的 avatar。一局共创活动 = N 个真人 + N 只卡皮同时在场，每只卡皮都有自己的行为和表达。

### 9.2 空间即场地

共创活动发生在某位玩家的河岸上，该玩家是"东道主"。这强化了空间的社交资本属性——"我家能承办的活动更丰富"变成一种吸引力。

### 9.3 架构的三层分离

V2 引入时，产品从纯异步升级为混合模式：

```
常驻层（异步）：空间、探索、记忆匹配、1v1 聊天 —— 治愈感主场
活动层（同步）：共创房间 —— 主动选择进入，预期投入时间
```

两层心理契约严格隔离——活动层的战绩/排名不污染家园。

### 9.4 V1 必须预留的技术/设计债

| 预留项 | V1 做什么 | V2 依赖 |
|--------|----------|--------|
| 长期对话历史访问 | AI 能读取长上下文（对话 + 记忆 + 行为轨迹）| 驱动共创时的卡皮个性差异 |
| 空间视角与尺度 | 选择能容纳 8+ 角色的视角基线 | 多人同屏 |
| 物品分类 tag | 五类结构先建，内容只填三类 | 场地道具与共创产出顺利接入 |
| 卡皮行为动词 | V1 基础动词集（explore / chat / recall / initiate / rest / swim / idle / eat / gaze / encounter），架构可扩展 | V2 新增 visit（串门）/ play_game / create / perform 等 |
| 关系系统 | 数据结构原生支持 N 人 graph | 团体关系自然承接 |
| 同步通信层 | 服务端架构预留（不需要 V1 就上 WebSocket，但不走死路） | 共创房间的实时协作 |

### 9.5 UGC 能力分层（V2 内部也有阶段）

| 阶段 | 活动类型 | 创作难度 |
|------|---------|---------|
| V2 早期 | 规则驱动（抓鱼大赛、投喂、抽卡） | 零门槛 |
| V2 中期 | 轻创作（合奏预制乐句、涂鸦贴纸、拍照合影） | 低门槛，组合式 |
| V2 远期 | 深创作（自由绘画、自由作曲）、推理类（狼人杀、剧本杀） | 高门槛，模型能力达标后再做 |

**特别注意**：狼人杀等强推理游戏依赖 AI 模型在社交博弈上的能力，目前尚未成熟。放到 V2 远期。

### 9.6 关于"卡皮是否会背叛你"的哲学问题

V2 时需要明确：玩家能多大程度引导自己卡皮在共创活动中的行为？

推荐方向：**"会听但不一定照做"**——玩家战前可以交代卡皮，卡皮决定听不听。这保留独立性同时给玩家情感投射空间。

这与 V1 日常对话里"卡皮对你话的响应程度"是**同一个原则的两个场景**：V1 日常对话、V1 出门意愿、V2 共创行为，都由 AI 基于对话历史理解的"会听但不一定照做"自然表达。V1 就要把这个基调立起来。

## 十、V1 范围清单

MVP 必做功能（精简版）：

```
【核心循环】
1. 养卡皮 + 日常对话（含卡皮主动起话题、情绪起伏）
2. 卡皮自主决定是否出门探索（70/30），单一时长档
3. 旅途中用户可随时主动发消息，卡皮秒回（内容体现"在路上"）；卡皮不主动推送
4. 归来见闻 + 整合全程 + 带回物品

【记忆匹配】
5. 后台记忆打标 + 向量化
6. 出门匹配一次相遇，回来自然生成见闻中的他人故事（主题级）
7. 匹配结果可点入对方河岸

【空间】
8. 河岸家园展示（视角和尺度为 V2 多人场景预留）
9. 家园装饰 / 物品摆放

【社交】
10. 对方河岸浏览
11. 留言搭话（异步）
12. 1v1 消息

【卡皮与意愿】
13. 卡皮个性由 AI 读对话历史自然生成（不做结构化性格数据）
14. 卡皮意愿行为词汇（婉拒、主动起话题、主动回忆、情绪反馈等）由 AI 判断触发
15. 生活层动作循环（4-5 个日常动作：睡/游/发呆/吃/看风景）
16. 卡皮装扮槽位系统（内容极简）+ 装扮情绪反馈
17. 新用户起始装扮赠送（V1 无先天变异，装扮是个体识别的唯一视觉来源）

【隐私与记忆】
18. 卡皮记忆库 UI（时间线 + 分享/私密切换 + 删除）
19. AI 自动分档 shareable/private
20. 敏感话题分支处理
```

**V1 明确不做的**：

- 卡皮主动串门访问他人河岸（V2）
- 长途冒险、探索多档时长（V2）
- 跨天旅行、多段式行程（V2）
- 商业化（V2）
- 显性等级、能力解锁（V2）
- 无主河岸 / 系统填充河岸（V2 或永不做，依 V1 验证结果）
- **装扮的拒绝穿脱**（V1 只做情绪反馈，不做抗拒）
- **对话层的冷漠/已读不回**（永不做，伤治愈感）
- 发现页 / 精选列表（永不做——破坏匹配驱动的纯度）
- 共创活动（V2）

## 十一、AI 的角色

AI 不是这个产品的卖点，产品体验才是。AI 在以下地方自然适配：

| 应用场景 | 作用 | 用户感知 |
|---------|------|---------|
| 伙伴对话 | 日常陪伴、有自己的性格和表达方式 | "它好有趣" |
| 对话驱动旅行 | 理解对话内容，影响地点选择和旅行倾向 | "我随口一说它真去了那种地方" |
| 地点选择 | 基于近期对话意向从地点库选当次旅行目的地 | "它今天去了京都！正好我最近想看樱花" |
| 旅途对话 | 用户主动发起时，每条回复内嵌当下地点情境 | "它真的在京都啊" |
| 意愿决策 | 根据性格+地点+累积互动，决定旅行偏好/情绪/主动性 | "它今天有点蔫" |
| 个性表达 | AI 读对话历史理解卡皮性情，在所有输出中保持一致 | "它越来越像它自己了" |
| 记忆结构化与分档 | 对话内容打标、嵌入、shareable/private 判定 | 记忆库可见；卡皮口头表达"我想记住"/"就你我知道" |
| 敏感话题识别 | 触发关怀分支、引导专业资源、阻止进入匹配池 | "它好像在认真地关心我" |
| 脱敏流水线 | 匹配共享前去除明确实体 | 无感知 |
| 匹配生成 | 在同地点池中按 A/B/C 档位选出相遇对象 | "今天它遇到的这个人，太懂我了" |
| 每日手记生成 | 图文结构、情境叙事、匹配事件自然融入、回响用户旅途问过的事 | "今天的手记又好看又好读" |
| 手记插图生成 | AI 生成图像（基于风格参考 + 地点 + 当日叙事） | "这图好看到想存下来" |
| 生活层动作调度 | 休息日根据性格/时段选择卡皮此刻的动作循环 | "它今天一直在发呆，好治愈" |
| 空间/物品生成 | 基于旅行结果程序化生成河岸装饰 | "我的河岸好好看" |

## 十二、美术方向（待定义）

美术是这个产品的**核心产品力**，不是锦上添花。用户截图发朋友圈的前提是"这个画面值得发"。

### 12.1 需要明确的方向

| 维度 | 候选方向 | 建议 |
|------|---------|------|
| 整体风格 | 水彩插画 / 像素风 / 扁平插画 / 3D 渲染 | 推荐水彩插画或精致扁平，治愈感最强 |
| 卡皮巴拉表情动作系统 | 极简（旅行青蛙式）/ 丰富（宠物养成式） | **必须丰富**——承载对话情感 + 性格表达 + 休息日 4-5 种动作循环 |
| 休息日生活层动作循环 | 睡觉 / 游泳 / 发呆 / 吃草 / 看风景（V1 至少 5 种） | 每种都要有独立动画，是 P0-C 假设验证的基础工作量 |
| **手记形态** | 图文混合 / **叙事漫画** | **V1 选择：叙事漫画**（竖屏 webtoon 格式，4-8 格/篇）。图承担主要叙事负担 |
| **漫画生成方案** | 纯 AI / 预制风格 + AI / 全手绘 | **V1 选择：预设风格参考 + AI 生成（含角色一致性控制）**。前期出画风定义包 + 卡皮形象锚定 + 旅伴卡皮通用形象 + 地点视觉包 |
| 角色一致性 | 同篇漫画内卡皮形象必须一致；跨日旅行内高一致 | 技术方案：IP-Adapter / ControlNet / LoRA 等（工程决策）|
| 地点美学定义 | 每个地点的视觉关键词、参考图、风格调性 | 350-600 个地点每个需要 3-5 张参考关键画面 |
| 家园装饰风格 | 写实自然 / 童话幻想 / 混合 | 需要用户测试 |
| 空间视角 | 俯视 / 45°斜视 / 第一人称逛 | **V1 选择必须能承接 V2 多人场景**（推荐 45°斜视） |

### 12.2 美术原则

- **截图即传播**——任何一帧画面都要值得发朋友圈
- **治愈大于精致**——温暖柔和的色调，避免冷硬的科技感
- **卡皮巴拉是视觉中心**——所有场景设计服务于突出卡皮巴拉
- **生活层动作必须"有灵魂"**——打开 app 看到它发呆，用户要觉得"好治愈"而不是"就一个循环动画"。这比做多个动作更重要
- **空间要有"可读性"**——别人被匹配进来后，能一眼看出你的记忆/经历是如何呈现的
- **V1 视角为 V2 多人场景留余地**——不要选择只能单人展示的视角

## 十三、MVP 验证策略

### 13.1 最需要验证的假设（按优先级排序）

**P0-A：对话驱动探索的可玩性**

用户跟卡皮说不同的话，探索结果的差异度能否做到足够大？更关键：用户能否**感知到**这种差异？

验证方式：纯对话+探索的原型，让用户试玩一周，观察：
- 对话内容的多样性
- 探索结果的惊喜度
- 用户是否能意识到"我的话影响了探索"

**P0-B：匹配段落的内容质量能否支撑社交转化**（与 P0-A 并列）

被匹配到的用户，读手记中匹配段落时，情感反应是否真的会被击中？用户读完后，有没有**真的想点入对方河岸**？这是整个社交模型的地基。

**匹配段落质量 = 社交漏斗的命门**：段落写得平，用户读完无感，不点入对方河岸 → P0-B 验证直接失败 → 社交模型不成立。所以 P0-B 本质上是"匹配段落的内容质量能否支撑转化"的测试，不是"匹配算法是否精准"的测试。

验证方式：
1. **wizard-of-oz 实验**：人工配对 5-10 组用户，给他们呈现人工撰写的优质匹配段落，观察：
   - 情感反应强度（感动？警惕？无感？）
   - 是否点入对方河岸
   - 是否留言
2. **AI 生成版对比**：同等情境下 AI 生成 prompt 产出的匹配段落与人工段落对比，评估差距
3. **内容质量批次审核**：上线前人工评估至少 30 篇 AI 匹配段落，"是否想点入对方河岸"命中率应 ≥ 70%

匹配段落的完整质量标准见 `02_记忆匹配系统.md` 第 4.5 节。

**P0-C：独立宠物感是否成立**（v2-update 新增，与 P0-A/B 并列）

用户使用两周以上，是否真正感到卡皮是一个**独立的存在**，而不是百依百顺的陪伴工具？这是本产品相对于所有 AI 陪伴产品的根本差异——不验证这个，伪人感的结构性问题依然存在，长期留存会复现 Character.ai 式的塌方曲线。

验证方式：
- 用户自发语言分析：是否出现"它今天有点蔫"、"它好像不想理我"、"它好贪吃"等人格化描述
- 代词使用观察：用户描述卡皮时用"它"还是"这个 app"
- **抗伪人感压力测试**：两周以上使用后，是否出现"突然觉得空虚/自欺欺人"的弃用信号？用户访谈聚焦这个感受
- 生活层的"被动观看时长"：用户打开 app 不聊天、不出门，只看卡皮在做什么的时长占比

这个假设的验证周期比 P0-A/B 长（至少 2-3 周），需要提前规划。

**P1：空间作为"记忆之家"的可读性**

当用户因为匹配而进入对方河岸时，能否从空间里读出那段被共振的记忆？空间是否让人想留言？

验证方式：给测试用户看几个不同的空间（对应不同记忆主题的呈现），观察他们的阅读体验和提问意愿。

**P2：从匹配到搭话的转化率**

匹配触发了用户的情感共鸣后，有多少人会实际点入对方河岸？其中又有多少会留言？

验证方式：原型中埋点观察行为漏斗。

### 13.2 冷启动策略

早期用户少时：

- **邀请制内测**，控制早期用户规模以保证每个人的空间都能被匹配到
- 手工运营前 20-50 位用户（包括引导他们分享足够的记忆素材，人工检查匹配质量）
- **不主动生成无主河岸**——一旦被用户发现是 AI 生成，本产品核心机制的信任崩塌
- 真实用户密度不够时，**宁愿降低匹配频率**（今天卡皮没遇到合适的人，空手而归），也不伪造匹配
- 后续密度起来后，匹配频率自然提升

#### "空手而归"的日子怎么办

早期可能 60-80% 的日子匹配不到合适的人。这不能让用户觉得产品"没内容"——**归来见闻必须独立成立**：

- 见闻主体始终是卡皮自己的旅途故事（发现、天气、景色、遇到的小动物、吃到的好吃的）
- 匹配到的"他人故事"是**加分项而非必需项**——有就让见闻变惊喜，没有就纯粹是旅途日记
- 用户预期引导：首次引导里明确"卡皮不是每天都会遇到投缘的朋友"，把匹配设为**惊喜**而非**日常**
- 卡皮在没匹配的日子可以主动说"今天没遇到什么特别的人，但是看到了好多小青蛙"——让"没匹配"本身成为内容的一部分
- 匹配频率相关的期望管理文案必须在 prompt 工程和 UI 文案中同步设计

这其实也是设计的**护城**：匹配不是廉价的——它被珍贵地对待，用户才会真正珍视每一次相遇。

## 十四、商业化（V2）

**V1 完全免费，不引入任何付费点。** 治愈感的验证不能被氪金压力污染。

### 14.1 V2 付费点

| 类型 | 说明 | 示例 |
|------|------|------|
| 空间装饰 | 购买更多装饰物和风格 | 樱花树、温泉灯、限定主题 |
| 伙伴装扮 | 商店装扮（与探索装扮视觉相同但无来历） | 围巾、花环、墨镜 |
| 场地道具 | 共创活动所需的独特场地物件 | 钢琴、画架、茶室 |
| 稀有物品 | 探索中的稀有掉落加速 | 限定收藏品 |

### 14.2 商业化原则

**花钱买的是"更多表达"，不是"更快进度"、"更多匹配"、"更深关系"。**

社交核心链路绝对不能付费：

- 匹配频率不能花钱提升
- 见闻质量不能花钱优化
- 记忆共鸣度不能花钱增强
- 留言成功率不能花钱干预

只要触碰上述任何一条，产品的信任根基会瞬间崩塌。

- 免费用户的空间也要好看
- 付费用户的空间只是"更有个性"
- 参考《动物森友会》的思路：付费拓展表达空间，而非制造免费用户的痛苦

## 十五、隐私与伦理

**这是本产品最需要长期投入的模块，不是一次性设计问题。**

### 15.1 核心策略：半透明 + 字段区分

不是"每条记忆都提醒用户确认"（太重，破坏对话流畅度），也不是"全量不透明使用"（信任风险高、法律站不住）。采用**半透明 + 字段区分**路线：

- 每条记忆在存储时带 `shareable: bool` 字段
- AI 自动预判初始值，敏感内容默认 `private`
- 用户可在"卡皮记忆库"随时查看、切换
- 只有 `shareable=true` 的记忆进入匹配池
- 首次注册时用一句柔软的话完成知情同意

### 15.2 首次知情同意文案（示例）

> "卡皮会记住你跟它分享的故事。有些故事它会带出去，在旅途中和其他卡皮交换——说不定能遇到和你有相似经历的人。当然，有些话它会替你藏着。你随时可以去它的小本子上，看看它都记住了什么。"

这段话：
- 主动说明"卡皮会带出去分享"，完成知情同意
- 主动说明"有些它会替你藏"，暗示存在分享/私密分档
- 指向"记忆库"入口，让用户知道可控
- 保留产品的柔软叙事，不像冷冰冰的隐私协议

### 15.3 AI 初始分档规则

AI 在打标时自动将以下类型记忆判定为 **private 候选**：
- 涉及家人/朋友的具体信息（姓名、职业、关系细节）
- 健康相关（疾病、用药、心理状态）
- 财务相关（债务、收入、消费具体数额）
- 法律相关（纠纷、案件、司法程序）
- 亲密关系具体细节
- 未成年人相关内容
- 用户明确说过"这个别跟别人说"的内容

判定 **shareable 候选** 的类型：
- 偏好、兴趣（喜欢什么颜色、什么食物、什么天气）
- 情绪状态（今天开心、难过、焦虑，但不涉及具体人名事件）
- 童年经历的主题（但不含具体身份信息）
- 对某类事物的态度、观点
- 一般性的生活方式描述（独居、夜猫子、宠物爱好）

**所有自动判定都可被用户手动翻转。**

### 15.4 "卡皮记忆库" UI（V1 必做）

这是用户能控制隐私的唯一窗口。必做功能：

- 时间线展示所有被卡皮记住的记忆（原话 + AI 归纳的主题）
- 每条记忆右上角有 🔓（shareable）/ 🔒（private）切换标签
- 可以批量筛选查看（"看看卡皮会分享哪些"、"看看它替我藏着哪些"）
- 每条可删除（从匹配池和卡皮对话上下文中一并清除）

记忆库的视觉呈现不要搞得像后台管理页。建议做成"卡皮的小本子"这种温暖的视觉语言。

### 15.5 已识别的风险

**🟡 技术风险：脱敏是一个难题**

即使 `shareable=true` 的记忆也要过脱敏流水线（去除明确实体）。但 AI 脱敏无法脱敏情感指纹——"我妈去年春天查出乳腺癌"就算去掉"妈"和"春天"，留下的信息量可能让熟人一眼认出。

需要一整套评估与红蓝军机制。建议 V1 上线前做至少一轮内部渗透测试，模拟熟人场景下的身份推断。

**🟡 分类错判风险**

AI 预判的 shareable/private 不可能完全正确。错判为 shareable 会导致隐私泄露，错判为 private 会让匹配池质量下降（对用户体验影响小）。**宁可多误判为 private**——用户在记忆库主动打开的成本可控，而泄露是不可逆的。

**🟡 合规风险**

《个人信息保护法》对"将个人信息用于匹配、推荐"要求单独同意。半透明路线 + 清晰的记忆库 + 知情同意文案，基本能满足合规要求。但建议上线前做一次专项合规评估。

### 15.6 敏感话题处理（V1 必做）

卡皮和用户聊天时可能触及敏感内容。需要明确：

- **自杀/自伤/严重心理危机**：卡皮表达关心（"听起来你很辛苦"），引导至专业援助资源（心理热线、紧急联系），绝不做医疗/心理建议。这类记忆自动判为 private 且不进入匹配池
- **暴力/犯罪内容**：卡皮不跟进、不讨论，引导至适当资源
- **未成年人相关**：触发特殊对话分支
- **医疗/法律咨询类**：卡皮明确表达"这个我帮不了你"，引导至专业渠道

这是 V1 前必须和合规、伦理顾问共同设计的模块。

### 15.7 其他硬规则

- 用户必须能随时导出自己的全部记忆数据
- 用户注销账号时，记忆数据硬删除（包括已脱敏共享出去的内容也要从匹配池清除）
- 敏感话题类记忆**永不进入匹配池**，无论 shareable 字段是什么
- 匹配到的结果用户可以"标记不感兴趣"，降低同类主题未来匹配频率
- 留言不受欢迎时，对方可一键静默（不通知本人，避免挫伤）

## 十七、竞争壁垒

| 壁垒类型 | 强度 | 说明 |
|---------|------|------|
| 交互范式先发 | 强 | "记忆驱动匹配 + 卡皮作为翻译者"如果率先定义，认知先发优势极强 |
| 记忆数据飞轮 | 强 | 用户越分享，匹配越精准；迁移无法带走积累的记忆素材 |
| 用户空间资产 | 中强 | 用户积累的独特空间和探索历史不可迁移 |
| 社交关系网络 | 强 | 用户之间建立的 1v1 深度关系是最强的迁移壁垒 |
| UGC 生态（V2） | 强 | V2 共创内容形成独特社区 |
| AI 对话与匹配数据 | 中 | 对话越多，探索和匹配的质量越高 |

**核心风险**：

1. 如果产品验证成功，大厂能快速复制玩法，但**记忆共振匹配的调优 know-how** 是难复制的部分
2. 隐私伦理争议如果先引爆，会直接定义这个赛道的天花板——必须主动做到伦理领先而不是被动合规
3. 长期壁垒依靠社交关系网络 + 社区氛围 + 伦理口碑的先发积累

## 十八、关键设计决策总结

| 决策 | 选择 | 原因 |
|------|------|------|
| AI 角色定位 | 独立伙伴，不是 AI 分身 | 避免恍惚感和伪人感 |
| 核心社交机制 | **地点 + 记忆双维度匹配**（A/B/C 三档） | 同地点偶遇 + 记忆共振双信号，覆盖面和深度兼得 |
| 旅行地点 | 真实世界地点，V1 预制 350-600 个地点库 | 具象感强于虚构；地点是社交的共同情境 |
| 旅行时长 | 1-5 天不等随机，多日旅行；休息日 1-2 天 | 贴合真实旅行感；每日手记节奏；休息日是生活层的舞台 |
| 每日推送 | 每晚 20:00 ± 30 分钟一次，推送当日手记；21:30 后不推送 | 一天一次是期待，超过就是打扰 |
| 手记形式 | **叙事漫画（竖屏 webtoon）**，每日一篇 4-8 格 + 80-180 字旁白 | 图承担主要叙事负担；传播性极强（天然适合社交分享） |
| 漫画文字处理 | **图内绝无任何文字**（AI 处理文字不稳定），所有旁白以独立文字段穿插在格子之间 | 避开 AI 生图的文字乱码问题；排版由产品侧完全控制 |
| 漫画生成 | **预设风格参考 + AI 生成 + 角色一致性控制** | 成本与一致性平衡；不需要海量手绘预制 |
| 匹配算法思路 | **先配对后定地点**：每天全局配对所有旅行用户，每对送到同一地点共同旅行 | 浪漫叙事"地点由缘分决定"；算法简单；每天必有匹配 |
| 判断逻辑原则 | **能交给 AI 的判断就交给 AI，不写硬编码映射表/权重公式** | 除了安全约束和硬产品参数，所有内容生成、性格表现、匹配复核、分档判断都由 AI 语义理解；调优改 prompt 不改代码 |
| 匹配相遇呈现 | **不分档位**，所有相遇都基于记忆匹配；深度由 AI 按相似度在连续光谱上自然呈现 | 避免梯度僵硬；AI 生成能力足够处理连续深度 |
| 漫画格数策略 | 4-8 格，由 AI 按相似度和叙事需要决定 | 匹配深度决定叙事长度 |
| "了解更多"升级入口 | 每篇手记下方都有"想多了解 Ta 一点吗？"按钮，进对话看卡皮讲浅层观察 | 给用户主动升级的通道；连接相遇和深度了解 |
| 发现入口 | 只来自卡皮的相遇报告 | 保证社交驱动力的纯度，避免变成刷人列表 |
| 产品入口 | 养成游戏，不是 AI 陪伴 | 解决"为什么要用你"的冷启动问题 |
| 卡皮独立性（V2） | 一起打狼人杀时是独立玩家而非 avatar | 兑现"不是分身"的产品承诺 |
| 空间定位 | 记忆的外化之家，不是社交引流页 | 深度大于广度 |
| "和别人的卡皮直接聊天" | 砍掉 | 伪人感风险；让卡皮只作为记忆翻译者，角色更清晰 |
| 物品交换系统 | 不做 | 交换的是经验和信息，不是物品；避免氪金感；纪念品永不可买 |
| 装扮 | 双轨（探索赚 + V2 商店购），槽位 V1 定死 | 兼顾独特性和商业化 |
| 卡皮个性 | **AI 读对话历史自然生成**（不做结构化性格数据） | 避免机械化；调优改 prompt 不改代码；个性是相处出来的 |
| 卡皮意愿表现 | AI 基于情境和对话历史判断各种意愿行为（婉拒、主动起话题等） | 不做分层机制、不做映射表；交给 AI |
| 探索触发 | 卡皮自主决定（70/30），用户不能强制 | 旅行青蛙式不可控感 + 独立感 |
| 生活层动作 | V1 必做 4-5 种日常动作循环 | 打开 app 的非交互动机，最直接的"它存在"视觉证据 |
| 空间视角 | V1 选择必须承接 V2 多人场景 | 视角 V1 定了 V2 改不了 |
| 用户优先级 | 女性优先 | 社交产品铁律 |
| 对外定位 | 治愈养成，不叫社交 App | 降低下载心理门槛 |
| 上手门槛 | 极致低门槛 | 不需要上传资料，打开即玩 |
| 旅途沟通方向 | 仅用户主动发起（除每晚一次手记推送外） | 不打扰是治愈感底线；主动权完全在用户 |
| 旅途对话响应 | 秒回，通过对话内容体现当下地点情境 | 对话流畅性优先；时空感靠内容不靠延迟 |
| 全天系统推送 | 只有一次——每晚的今日手记 | 情感积累的兑现，不是打扰 |
| 商业化 | V1 完全免费，V2 才引入 | 治愈感和氪金压力不同时引入 |
| 商业化原则 | 买表达不买进度、不买社交关系 | 保护信任根基 |
| 隐私策略 | 半透明 + 记忆字段区分（可分享/私密）+ 脱敏 + 记忆库可控 | 平衡神秘感、信任、合规、用户掌控感 |
| 冷启动 | 邀请制 + 手工运营，不做无主河岸 | 保护机制信任，宁可慢 |
| 卡皮生命周期 | 不老、不病、不死，陪伴用户长期 | 避免情感伦理复杂度；旅行青蛙路线；机制简洁 |
| 性格重塑 | 不提供显式重置，只靠日常相处自然漂移 | 真实宠物不可重养；避免"重抽卡"破坏独立角色定位 |

## 十九、待深入定义的领域

按 V1 上线前优先级：

### V1 必须定义（P0）

1. **对话 → 探索的映射规则** — 对话内容如何转化为探索路径？Prompt 工程方案
2. **记忆打标与向量化方案** — 哪些对话内容进入匹配池？怎么打标签？向量模型选型
3. **匹配算法 v0** — MVP 用什么相似度计算？冷启动时匹配池不足怎么办？
4. **记忆分档规则** — AI 预判 shareable/private 的具体规则；记忆库 UI 的视觉和交互；与意愿记忆层的双面实现
5. **脱敏流水线** — 技术方案 + 红蓝军测试机制
6. **归来见闻生成 prompt 工程** — 在见闻中自然带出他人主题、不露细节、整合旅途全程对话、回响用户主动问过的事
7. **旅途对话 prompt 工程** — 每条用户问询的回复都要内嵌时空感；用户问询如何影响路径权重
8. **敏感对话处理流程** — 触发词、处理策略、法律咨询
9. **卡皮个性 prompt 模板** — AI 在任何生成任务中都读取的"卡皮性情理解"片段，如何从对话历史中精炼性情特征
10. **意愿行为词汇的 prompt 工程** — 婉拒、主动起话题、主动回忆、表达记忆判断等行为的触发和表达原则
11. **生活层动作设计** — 动作清单 + 切换逻辑 + 与性格/时段的关系 + 美术规格
12. **空间视角与尺度基线** — 美术概念稿验证，V2 多人场景可行性
13. **家园物品与装扮库的 MVP 内容** — 至少各 15-20 款可用
14. **P0-C 验证方法论** — 独立宠物感如何量化？用户访谈脚本 + 行为埋点设计
15. **数据指标体系** — P0-A、P0-B、P0-C、P1、P2 假设的具体量化指标
16. **冷启动匹配池兜底** — 用户稀少时卡皮"空手而归"的见闻内容设计 + 用户期望管理

### V1 可以后定（P1）

17. **卡皮美术表情动作系统的完整库** — V1 先够用即可，后续扩展

### V2 阶段才要定义

18. **卡皮主动串门算法** — 从匹配走向推荐
19. **共创活动设计** — 从轻共创到深共创的具体玩法
20. **同步通信层架构** — WebSocket / 长连接方案
21. **团体关系与权限系统** — 多人关系图谱
22. **商店与氪金策略细节**
23. **装扮的拒绝穿脱机制** — 从 V1 情绪反馈升级为抗拒
24. **UGC 审核机制** — V2 共创内容的安全
