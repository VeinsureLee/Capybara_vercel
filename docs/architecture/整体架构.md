# 整体架构

> 技术栈、架构分层、目录结构

---

## 一、技术栈总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Next.js 15 (App Router) + React 19 | SSR/SSG 混合、自动代码分割 |
| 语言 | TypeScript (strict mode) | 全栈类型安全 |
| 样式 | Tailwind CSS v3 + 自定义动画 | 原子化 CSS + 12 组关键帧动画 |
| 后端 | Next.js API Routes (Serverless) | Vercel Edge/Serverless Functions |
| 数据库 | Supabase PostgreSQL | RLS + Realtime + Auth 一体化 |
| 认证 | Supabase Auth | Email/Password + OAuth（可扩展） |
| AI 服务 | DeepSeek / Claude / OpenAI | 三级降级 + 本地模板兜底 |
| 部署 | Vercel | 零运维、自动 CI/CD |

## 四、目录结构

```
src/
├── app/                          # Next.js App Router 页面
│   ├── layout.tsx               # 根布局（字体、全局样式）

│   ├── globals.css              # 全局样式 + 12 组 CSS 动画
│   ├── login/page.tsx           # 登录
│   ├── register/page.tsx        # 注册
│   ├── chat/page.tsx            # 对话主界面
│   ├── home/page.tsx            # 河岸空间
│   ├── explore/page.tsx         # 探索页面
│   └── api/
│       ├── auth/callback/route.ts  # OAuth 回调
│       ├── capybara/route.ts       # 卡皮巴拉 CRUD
│       ├── chat/route.ts           # 对话处理
│       └── explore/route.ts        # 探索生成/查询
├── components/
│   ├── BottomNav.tsx            # 底部导航栏
│   └── explore/                 # 探索动画组件群
│       ├── ExploreScene.tsx     # 场景管理器
│       ├── DepartureStage.tsx   # 出发阶段
│       ├── JourneyStage.tsx     # 旅途阶段
│       ├── DiscoveryStage.tsx   # 发现阶段
│       ├── ReturnStage.tsx      # 归来阶段
│       ├── ProgressBar.tsx      # 进度条
│       └── FloatingElements.tsx # 浮动元素
├── lib/
│   ├── ai/
│   │   ├── client.ts           # 多 AI 提供商客户端
│   │   └── prompts.ts          # 提示词模板

│       ├── client.ts           # 浏览器端 Supabase

├── types/
│   └── index.ts                # 全局类型定义
└── middleware.ts                # 认证中间件
```

## 五、环境变量清单

| 变量名 | 必需 | 暴露给前端 | 说明 |
|-------|------|----------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | 是 | 是 | Supabase 项目 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 是 | 是 | Supabase 匿名 Key |
| `DEEPSEEK_API_KEY` | 否 | 否 | DeepSeek AI（推荐） |
| `CLAUDE_API_KEY` | 否 | 否 | Claude AI |
| `OPENAI_API_KEY` | 否 | 否 | OpenAI |

> 至少配置一个 AI API Key 才能获得 AI 回复。无 Key 时使用本地模板兜底。
