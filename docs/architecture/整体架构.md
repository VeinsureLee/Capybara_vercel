# 整体架构

> 技术栈、架构分层、目录结构

---

## V2 设计理念

V2 将产品从「关键词驱动探索 → 抽象空间差异 → 好奇心社交」迁移到「记忆驱动旅行 → 真实地理位置 → 记忆共鸣社交」。核心变化：

- **记忆系统**：对话 → AI 提取结构化记忆 → shareable/private 标记 → 可分享记忆进入全局匹配池
- **多日旅行**：1-5 天真实地点旅行（全球 350-600 个地点，MVP ~60 个），替代 V1 小时级探索
- **每日手记**：Webtoon 风格叙事漫画，偶遇自然融入叙事
- **记忆驱动匹配**：全局每日批量匹配，按记忆相似度配对，配对用户的卡皮巴拉前往同一地点
- **卡皮巴拉自主意志**：~10% 拒绝率，休息日执行生活层动作（sleep/swim/idle/eat/gaze）
- **装扮系统**：4 槽位（头/身/尾/配饰），双轨获取（探索获得 + V2 商城）
- **隐私边界**：记忆 shareable/private 分类，AI 自动标记，用户可控记忆库
- **商业化边界**：V1 完全免费，V2 引入付费装扮道具，社交功能永远免费

## 一、技术栈总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | Next.js 15 (App Router) + React 19 | SSR/SSG 混合、自动代码分割 |
| 语言 | TypeScript (strict mode) | 全栈类型安全 |
| 样式 | Tailwind CSS v3 + 自定义动画 | 原子化 CSS + 12 组关键帧动画 |
| 后端 | Next.js API Routes (Serverless) | Vercel Edge/Serverless Functions |
| 数据库 | Supabase PostgreSQL | RLS + Realtime + Auth 一体化 |
| 认证 | Supabase Auth | Email/Password + OAuth（可扩展） |
| AI 服务 | DeepSeek / Claude / OpenAI | 三级降级 + 本地模板兜底 |
| 部署 | Vercel | 零运维、自动 CI/CD |
| **V2 新增：记忆提取** | AI 结构化提取 + 隐私标记 | `src/lib/memory/extract.ts` |
| **V2 新增：旅行匹配** | 记忆相似度 + 地点分配 | `src/lib/travel/matching.ts` |
| **V2 新增：地点目录** | 全球真实地点 (MVP ~60) | `src/lib/travel/locations.ts` |

## 四、目录结构


## 五、环境变量清单

| 变量名 | 必需 | 暴露给前端 | 说明 |
|-------|------|----------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | 是 | 是 | Supabase 项目 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 是 | 是 | Supabase 匿名 Key |
| `DEEPSEEK_API_KEY` | 否 | 否 | DeepSeek AI（推荐） |
| `CLAUDE_API_KEY` | 否 | 否 | Claude AI |
| `OPENAI_API_KEY` | 否 | 否 | OpenAI |

> 至少配置一个 AI API Key 才能获得 AI 回复。无 Key 时使用本地模板兜底。
