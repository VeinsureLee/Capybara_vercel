# 技术债务与环境变量

> 已知技术债务清单与环境变量配置。

---

## 十四、已知技术债务

| 问题 | 优先级 | 建议修复 |
|------|-------|---------|
| 无 API 输入验证 | P1 | 添加 zod schema 验证 |
| 无 Rate Limiting | P1 | Vercel Edge Middleware 或 Upstash Redis |
| 无错误边界 | P2 | 添加 `app/error.tsx`, `app/not-found.tsx` |
| 前端状态不共享 | P2 | 引入 React Context + SWR |
| 无测试 | P2 | 配置 Vitest + 核心用例 |
| AI 响应无 streaming | P3 | 改为 SSE 流式响应 |
| 无日志系统 | P3 | 集成 Vercel Analytics + Sentry |
| crypto.randomUUID hack | P3 | 部署到 HTTPS 后可使用原生 API |
| 探索时长过短（MVP） | P3 | 上线后调整为产品文档设定值 |

## 十五、环境变量清单

| 变量名 | 必需 | 暴露给前端 | 说明 |
|-------|------|----------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | 是 | 是 | Supabase 项目 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 是 | 是 | Supabase 匿名 Key |
| `DEEPSEEK_API_KEY` | 否 | 否 | DeepSeek AI（推荐） |
| `CLAUDE_API_KEY` | 否 | 否 | Claude AI |
| `OPENAI_API_KEY` | 否 | 否 | OpenAI |

> 至少配置一个 AI API Key 才能获得 AI 回复。无 Key 时使用本地模板兜底。
