# 技术债务与环境变量

> 已知技术债务清单与环境变量配置。涵盖 V1 遗留债务与 V2 新增技术需求。

---

## 十四、已知技术债务

### V1 遗留债务

| 问题 | 优先级 | 建议修复 |
|------|-------|---------|
| 无 API 输入验证 | P1 | 添加 zod schema 验证 |
| 无 Rate Limiting | P1 | Vercel Edge Middleware 或 Upstash Redis |
| 无错误边界 | P2 | 添加 `app/error.tsx`, `app/not-found.tsx` |
| 前端状态不共享 | P2 | 引入 React Context + SWR |
| 无测试 | P2 | 配置 Vitest + 核心用例 |
| AI 响应无 streaming | P3 | 改为 SSE 流式响应 |
| 无日志系统 | P3 | 集成 Vercel Analytics + Sentry |
| crypto.randomUUID hack | P3 | 部署到 HTTPS 后可使用原生 API |
| 探索时长过短（MVP） | P3 | 上线后调整为产品文档设定值 |

### V2 新增技术债务

| 问题 | 优先级 | 建议修复 |
|------|-------|---------|
| 记忆提取 prompt 工程调优 | P0 | 设计 shareable/private 分档提取 prompt，反复测试边界 case（隐私泄漏、过度脱敏、主题丢失） |
| 脱敏流水线实现 | P0 | 将 V1 的 regex 黑名单升级为完整脱敏 pipeline（正则 + LLM 二次审查），覆盖记忆库与手记输出 |
| 敏感话题处理 | P0 | 实现 crisis 引导分支（自伤/危机关键词检测 → 温和中断 + 资源推荐），确保不生成有害内容 |
| 匹配算法 v0 | P0 | 实现记忆驱动匹配：`0.6·topic_sim + 0.25·emotion_sim + 0.15·intent_sim`，全局贪心配对，替换 V1 Jaccard 方案 |
| 旅行系统 → 替换探索系统的前端适配 | P1 | 多日旅行 UI 替换四阶段探索动画，适配 1-5 天旅行进度、旅途对话入口、归来见闻展示 |
| 手记生成 prompt 调优 | P1 | 叙事漫画结构 prompt 设计，包含匹配相遇段落融入、地点情境描写、角色一致性控制 |
| 记忆库 UI | P1 | 用户可浏览/管理记忆条目（shareable ↔ private 切换、删除），需设计可控的知情同意交互 |
| V1 探索系统清理 | P2 | 迁移 `explorations` 表数据至 `travels`，清理废弃的探索 API routes 与前端组件 |
| pgvector 向量化 | P2 | 为记忆条目生成 embedding 并存入 pgvector，支撑 topic_sim 计算与语义检索 |
| 实时推送手记 | P2 | 手记生成完成后通过 Supabase Realtime 或 SSE 推送至前端，替代轮询 |

## 十五、环境变量清单

| 变量名 | 必需 | 暴露给前端 | 说明 |
|-------|------|----------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | 是 | 是 | Supabase 项目 URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 是 | 是 | Supabase 匿名 Key |
| `DEEPSEEK_API_KEY` | 否 | 否 | DeepSeek AI（推荐） |
| `CLAUDE_API_KEY` | 否 | 否 | Claude AI |
| `OPENAI_API_KEY` | 否 | 否 | OpenAI |

> 至少配置一个 AI API Key 才能获得 AI 回复。无 Key 时使用本地模板兜底。
