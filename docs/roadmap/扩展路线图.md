## 移动端 App 打包方案

> 将 Next.js Web 应用打包为可安装的手机 App（APK/IPA）。核心挑战：项目依赖服务端 API Routes，无法纯静态导出。

### 背景约束

- 项目使用 Next.js App Router + 大量 API Routes（`/api/chat`、`/api/travel`、`/api/memory` 等）
- API Routes 依赖服务端运行时（Supabase 服务端客户端、AI 密钥等），不可打包进客户端
- 因此移动端 App **必须连接远程后端**（Vercel 部署），不支持纯离线运行

### 方案对比

| 方案 | 原理 | 优点 | 缺点 | 推荐场景 |
|------|------|------|------|----------|
| **A. Capacitor（推荐）** | 原生壳 + WebView 加载远程 URL | 可生成真 APK/IPA；可接入原生能力（推送、相机）；保留完整 Next.js SSR/API | 需 Android Studio 构建；本质仍是 WebView | 需要分发 APK + 后续接入原生能力 |
| **B. PWA** | Service Worker + Web App Manifest | 零改动成本；"添加到主屏幕"即用；支持离线缓存静态资源 | 不生成 APK；iOS 限制较多（无推送等） | 快速验证、内部测试 |
| **C. TWA** | 将 PWA 包装为 APK（Chrome Custom Tab） | 可上架 Google Play；轻量 | 仅 Android；需验证域名所有权 | 上架 Google Play |

### 推荐方案：Capacitor

#### 架构设计

```
┌─────────────────────────────────┐
│        Android / iOS 壳         │
│    (Capacitor Native Runtime)   │
│                                 │
│  ┌───────────────────────────┐  │
│  │       WebView             │  │
│  │  加载 Vercel 部署的 URL    │  │
│  │  https://capybara.vercel  │  │
│  └───────────────────────────┘  │
│                                 │
│  原生插件层（可选）：             │
│  - 推送通知                      │
│  - 状态栏控制                    │
│  - 闪屏/图标                     │
└─────────────────────────────────┘
         │
         │ HTTPS
         ▼
┌─────────────────────────────────┐
│      Vercel (远程后端)           │
│  Next.js SSR + API Routes       │
│  Supabase + AI Providers        │
└─────────────────────────────────┘
```

#### 实施步骤

**1. 安装依赖**

```bash
npm install @capacitor/core @capacitor/cli @capacitor/android --legacy-peer-deps
```

**2. 初始化 Capacitor**

```bash
npx cap init "Capybara" "com.capybara.app" --web-dir=out
```

**3. 配置 `capacitor.config.ts`**

```typescript
import type { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.capybara.app',
  appName: 'Capybara',
  webDir: 'out',
  server: {
    // 生产：指向 Vercel 部署 URL
    url: 'https://your-capybara.vercel.app',
    // 开发：指向局域网 IP（手机和电脑需同一 WiFi）
    // url: 'http://192.168.x.x:3000',
    cleartext: true, // 开发时允许 HTTP
  },
  android: {
    allowMixedContent: true, // 开发时允许混合内容
  },
};

export default config;
```

**4. 添加 Android 平台**

```bash
npx cap add android
```

**5. 构建 APK**

```bash
# 方式一：用 Android Studio 打开并构建
npx cap open android

# 方式二：命令行构建（需配置 Android SDK）
cd android && ./gradlew assembleDebug
# APK 产出路径：android/app/build/outputs/apk/debug/app-debug.apk
```

#### 前置要求

- **Android Studio**：用于构建 APK，需安装 Android SDK（API Level 22+）
- **Vercel 部署**：App 运行时需连接远程后端，因此必须先将项目部署到 Vercel
- **开发调试**：可临时将 `server.url` 改为局域网 IP，手机与电脑需在同一 WiFi

#### 开发 vs 生产配置切换

| 配置项 | 开发模式 | 生产模式 |
|--------|---------|---------|
| `server.url` | `http://192.168.x.x:3000` | `https://your-capybara.vercel.app` |
| `cleartext` | `true` | `false`（仅 HTTPS） |
| `allowMixedContent` | `true` | `false` |

#### 后续可扩展的原生能力

| 能力 | Capacitor 插件 | 用途 |
|------|---------------|------|
| 推送通知 | `@capacitor/push-notifications` | 手记生成完成、匹配成功提醒 |
| 本地通知 | `@capacitor/local-notifications` | 卡皮归来、休息结束提醒 |
| 相机 | `@capacitor/camera` | 用户头像上传 |
| 状态栏 | `@capacitor/status-bar` | 沉浸式 UI |
| 闪屏 | `@capacitor/splash-screen` | 启动画面 |
| 分享 | `@capacitor/share` | 手记分享到社交媒体 |

### PWA 方案（备选，可与 Capacitor 共存）

如果只需快速让用户在手机上体验，PWA 是零成本方案：

**1. 添加 `public/manifest.json`**

```json
{
  "name": "Capybara - 卡皮巴拉养成",
  "short_name": "Capybara",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#f0fdf4",
  "theme_color": "#8B6914",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

**2. 安装 `next-pwa` 并配置**

```bash
npm install next-pwa
```

**3. 用户在 Chrome 中打开 → 菜单 → "添加到主屏幕"**

> PWA 和 Capacitor 不冲突——PWA 提供即时可用的"类 App"体验，Capacitor 提供可分发的真 APK。建议两者都做。
